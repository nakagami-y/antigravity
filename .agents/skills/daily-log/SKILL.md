# デイリーログ（勤務報告）

1日の作業を「始め」と「終わり」で記録する。
plan/report が「タスク単位」なのに対し、こちらは「1日単位」の時間軸で記録する。

---

## トリガー

- `/daily-log` — 状況に応じて開始 or 終了を判定
- `/daily-log start` — 勤務開始（今日の予定を記録）
- `/daily-log end` — 勤務終了（やったことをまとめる）
- 「勤務報告」「今日の作業開始」「作業おわり」「日報」

---

## ワークフロー

### Step 0: モード判定

1. 引数に `start` / `end` があればそのモードへ
2. 引数なしの場合:
   - 今日のログファイルが **存在しない** → 開始モード
   - 今日のログファイルが **存在する** → 終了モード

判定基準: `memory_bank/report/{YYYY-MM-DD}_daily-log.md` の有無で判定する。

---

### 開始モード（start）

**ユーザーへの質問**（1問ずつ）:

1. 「今日やる予定のタスクを教えてください（箇条書きでOK）」
   - 自由入力

2. 「特に集中したいこと・ゴールはありますか？（なければスキップ）」
   - 自由入力 or スキップ

**生成アクション**:

- 前日のログがあれば Read して「昨日の明日やること」を引き継ぐ
- ログファイルを作成（開始セクションのみ）

**判断軸**: 前日からの繰り越しがある場合、「これ昨日の繰り越しだけど、今日もやる？」と確認する。惰性で繰り越し続けるタスクは「本当に必要か」を問う。

---

### 終了モード（end）

**ユーザーへの質問**（1問ずつ）:

1. 「今日やったことを教えてください（箇条書きでOK）」
   - 自由入力

2. 「予定と比べてどうでしたか？」
   - 選択肢: 予定通り / 予定以上にできた / 予定の一部ができなかった / 全然違うことをやった

3. 「今日の学び・気づきはありますか？（なければスキップ）」
   - 自由入力 or スキップ

4. 「明日やることを1つだけ決めるとしたら？」
   - 自由入力

**生成アクション**:

- 今日のログファイルに終了セクションを追記する
- 予定 vs 実績の差分を自動で整理する

**判断軸**:
- 予定と実績が大きくズレた場合 → なぜズレたか1行メモを促す（振り返りの材料）
- 3日連続で同じタスクが繰り越されている場合 → 「分割するか、やめるか」を提案する

## 出力フォーマット

```markdown
---
type: daily-log
date: {YYYY-MM-DD}
status: {open / closed}
---

# デイリーログ {YYYY-MM-DD}

## 開始（{HH:MM}）

### 今日の予定
- [ ] {タスク1}
- [ ] {タスク2}
- [ ] {タスク3}

### 昨日からの繰り越し
- [ ] {繰り越しタスク}（← {繰り越し元の日付}）

### 今日のゴール
{集中したいこと}

---

## 終了（{HH:MM}）

### やったこと
- [x] {完了タスク1}
- [x] {完了タスク2}
- [ ] {未完了タスク} → 明日に繰り越し

### 予定との差分
{予定通り / ズレた場合はその理由}

### 学び・気づき
{あれば記載}

### 明日やること
- [ ] {最優先タスク}
```
