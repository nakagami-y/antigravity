# スキル設計ガイド（スキルを作る・改修するときに参照）

スキル（`.agents/skills/*/SKILL.md`）を作るときの設計原則。
「ただの ToDo リスト」ではなく「判断力ごと再利用できるスキル」を作る。

## 核心

すべてのスキルは、具体層と抽象層の 2 層で設計する。

| 層 | 役割 |
|----|------|
| 具体層 | 何をやるか、どの順番でやるか |
| 抽象層 | どの視点で見るか、何を優先するか |

## 必須要素

1. トリガー
2. 入力定義
3. ワークフロー
4. 出力フォーマット
5. 保存先
6. セルフチェック

## 良いスキルの条件

- 曖昧な「いい感じに」を残さない
- 各 Step に OK / NG の判断軸がある
- 入力不足時の分岐がある
- 出力先パスが明記されている
- 最終セルフチェックがある
