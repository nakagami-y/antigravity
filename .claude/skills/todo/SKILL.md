# Todo管理

シンプルなタスク管理。1ファイル（`memory_bank/todo.md`）で全タスクを管理する。
複雑なプロジェクト管理ツールの代替ではなく、「今やるべきことを見失わない」ためのスキル。

---

## トリガー

- `/todo` — タスク一覧を表示
- `/todo add {タスク}` — タスクを追加
- `/todo done {番号}` — タスクを完了
- `/todo clean` — 完了済みをアーカイブ
- 「タスク追加して」「Todo見せて」「何やるんだっけ」

---

## ワークフロー

### Step 0: ファイル確認

1. `memory_bank/todo.md` を Read する
2. ファイルが存在しなければ初期テンプレートで作成する

### 引数なし → 一覧表示

現在のタスクを表示する。表示時の判断:

- **期限切れ** のタスクがあれば先頭に警告表示
- **7日以上動いていない** タスクがあれば「まだやる？」と確認
- タスクが **10個以上** あれば「多すぎないか？上位3つに集中しよう」と提案

```
📋 Todo（{N}件）

🔴 期限切れ:
  1. [!] {タスク} （期限: {日付}）

⬚ 未完了:
  2. {タスク}（優先: 高 / 期限: {日付}）
  3. {タスク}
  4. {タスク}

✅ 最近の完了（直近5件）:
  - {タスク}（{完了日}）
```

### `add` → タスク追加

**入力の解釈**:
- `/todo add レポートを書く` → タスク名のみ
- `/todo add レポートを書く p:高 d:2/20` → 優先度＋期限付き

引数がない場合は質問する:
1. 「何をやる？」（必須）
2. 「優先度は？」（選択: 高 / 中 / 低。デフォルト: 中）
3. 「期限はある？」（自由入力 or なし）

**判断軸**:
- タスクが曖昧（「あれやる」「考える」等）→ 具体化を促す（「何を、どこまでやったら完了？」）
- 既に似たタスクがある → 重複を指摘して確認

### `done` → タスク完了

- 番号を指定して完了にする
- 完了日を自動記録
- 完了時に「次に手をつけるべきタスクは？」と1つ提案する

### `clean` → アーカイブ

- 完了済みタスクを `## アーカイブ` セクションに移動する
- 30日以上前のアーカイブは削除する（ファイル肥大化防止）

## ファイルフォーマット

```markdown
---
type: todo
updated: {YYYY-MM-DD}
---

# Todo

## 未完了

- [ ] {タスク} | 優先: {高/中/低} | 期限: {YYYY-MM-DD or なし} | 追加: {YYYY-MM-DD}
- [ ] {タスク} | 優先: 中 | 期限: なし | 追加: {YYYY-MM-DD}

## 完了

- [x] {タスク} | 完了: {YYYY-MM-DD}
- [x] {タスク} | 完了: {YYYY-MM-DD}

## アーカイブ

<!-- 30日以上前の完了タスク。/todo clean で整理 -->
```

## 設計の判断軸

### なぜ1ファイルか

- 複数ファイルに分散すると管理コストが上がる
- Todoは「今何やるか」を即座に確認するもの。検索不要にする
- プロジェクト別の詳細管理が必要なら、それは `/plan` の仕事

### タスクの粒度

- **大きすぎるタスク**（「Webサイトを作る」等）→ 「最初の1ステップは何？」と分解を促す
- **小さすぎるタスク**（「メール1通返す」等）→ Todoに入れず今すぐやれ、と促す
- 目安: **30分〜3時間で完了する粒度**

### plan / daily-log との使い分け

| ツール | 単位 | 用途 |
|--------|------|------|
| `/todo` | タスク一覧 | 「次に何やる？」を見失わないため |
| `/plan` | 単一タスクの計画 | 「このタスクをどう進める？」の詳細設計 |
| `/daily-log` | 1日の記録 | 「今日何やった？」の振り返り |

**連携パターン**: `/todo` でやることを決める → `/plan` で計画を書く → 作業する → `/daily-log end` で1日を締める
