# フック生成（壁打ち・比較用）

「1行目で読むか読まないかが決まる」— すべてのコンテンツの生死はフックで決まる。
このスキルは感覚ではなく**心理メカニズム×型**の構造でフックを設計する。

---

## トリガー

- `/hook [トピック] [媒体]`
- 「フック作って」「フック案出して」「タイトル案出して」「見出し考えて」

## 入力

- **トピック**: 必須。フックを作る対象テーマ
- **媒体**: 必須。X / note / LINE / リール / TikTok / Shorts / YouTube / ブログ / 広告 / メール件名
- **ターゲット**: 任意。誰に向けたフックか（未指定なら汎用）
- **型の指定**: 任意。H1〜H8の指定（未指定なら自動選択）
- **証拠素材**: 任意。実績/事例/データ

---

## 5つの心理メカニズム（なぜフックが効くか）

| メカニズム | フックでの使い方 |
|-----------|---------------|
| **好奇心ギャップ** | 重要だが未完了の情報を提示し、即答しない |
| **損失回避** | 「避けたい損失」「やると損する落とし穴」を冒頭に |
| **未完了効果** | 結論の一部を見せて未完了状態を作る |
| **社会的証明** | 「X人がやっている」「事例の数字」を冒頭に |
| **処理流暢性** | 短い・具体的・平易・数字明確・動詞で動く |

---

## 8型フレームワーク（H1〜H8）

| 型ID | 名称 | 構造 | 主トリガー |
|------|------|------|----------|
| **H1** | ベネフィット先出し | {対象}は{方法}で{成果}できる | 処理流暢性 |
| **H2** | 痛み→回避 | {よくある失敗}、それ{原因}です | 損失回避 |
| **H3** | 好奇心ギャップ | {問い}の答え、{よくある誤解}です | 情報ギャップ |
| **H4** | 逆張り（反常識） | {常識}は{NG}、{理由}は{一言} | 注意喚起 |
| **H5** | 数字で具体化 | {時間/回数}で{成果}する{手順数} | 処理流暢性+具体 |
| **H6** | 証拠・事例 | {実例}で{結果}、やったことは{施策} | 社会的証明 |
| **H7** | 参加型（質問/診断） | {質問}。当てはまったら{次の一言} | 自己関与 |
| **H8** | 期限・希少 | {期限}までに{行動}しないと{損} | 損失回避+希少性 |

## 媒体別の最適設計

| 媒体 | 勝負する場所 | 相性の良い型 |
|------|-----------|-----------|
| **X** | 1行目の13文字 | H4, H5, H2, H7 |
| **note/ブログ** | タイトルが9割 | H5, H4, H3 |
| **LINE/メール** | 通知プレビュー20文字 | H1, H7, H8 |
| **短尺動画** | 0〜3秒（映像+字幕） | H1/H6, H3/H7 |
| **YouTube長尺** | タイトル/サムネ→冒頭30秒 | H1/H6, H2, H3 |
| **広告** | ヘッドラインのクリック | H1, H2, H5 |

## ワークフロー

### Step 1: 心理メカニズムの選定

媒体×トピックから、狙うべき心理メカニズムを特定する。
「なぜこのメカニズムか」を1行で述べる。

### Step 2: 型の選定（H1〜H8）

上の「媒体別の最適設計」を参照し、最適な型を最大3つ選定。
各型の選定理由を1行で述べる。

### Step 3: フック案10個の生成

選定した型を使い、フック案を10個生成する。
- 1つの型に偏らない（3型をバランスよく）
- 媒体の文字数/秒数制約を守る
- 証拠素材があれば活用する

### Step 4: 自己評価

各フック案に以下を付記:
- **使用型**: H1〜H8のどれか
- **狙う心理メカニズム**: 1つ
- **期待一致**: OK / 要修正（フックの約束を本文で回収できるか）
- **強度**: ★★★（親指が止まる）/ ★★☆（目が留まる）/ ★☆☆（流される）
- **KPI仮説**: 1行

### Step 5: A/B案の提示

10個から対照的な2案を選び、A/B案として提示する:
- A案: 最も引きが強いもの
- B案: Aとは異なる型（比較用）
- どちらをテストすべきか、理由とともに推奨
