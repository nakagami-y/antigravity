## 1. Content Decision Knowledge

### 1.1 Why Content Exists

- Content is not "posting" — it's a **device that moves people to the next step in the conversion route**
- 1 Channel = 1 Job: Each platform's only job is "send the reader to the next step"
- Content must deliver standalone value. 70-80% should be useful for free. CTAs only work when content earns trust first

### 1.2 Master Content Strategy

**Circulation Loop**: Collect raw material → Weekly triage → Create master content → Distribute across channels → Collect feedback → Back to raw material

**3-Tier Definition**:
| Tier | Frequency | Format | Distribution Power |
|------|-----------|--------|-------------------|
| Major Master | 1-2/week | Long-form articles (3,000-5,000+ words) / Video scripts / Webinar scripts | 10+ channels |
| Minor Master | 3-5/week | Newsletters / Client case summaries / Results reports | 3-5 channels |
| Raw Material | Daily | Work logs / Voice memos / Screenshots / Failure notes | Fuel for master content |

**Promotion Criteria (3+ of 6 = Major Master candidate)**:
1. You have first-hand experience
2. You have numbers/data (Before → After)
3. You have customer/user testimonials
4. You have a differentiation point vs competitors
5. It directly connects to a core business lever
6. It connects to a timely trend

**Strongest material source**: Client conversations + customer feedback. First-hand information competitors don't have.

### 1.3 Channel Role Map

| Channel | Role | Position in Conversion Route |
|---------|------|------------------------------|
| Twitter/X | Build trust — sharp takes, frameworks, contrarian views | Daily touchpoint |
| TikTok/Reels | Discovery — 45-second hooks that move people | Discovery |
| Instagram | Save-worthy content — checklists, templates | Daily touchpoint |
| YouTube | Long-term asset — structured tutorials, case studies | Trust building |
| Blog/Articles | Deep dives — failure stories, frameworks, templates | Trust building |
| Newsletter | Education — Why + one action today | Trust → Problem awareness |
| Community | Retention — weekly challenges, accountability | Ongoing engagement |
| Webinar/Event | Experience — education + offer | Experience |
| 1-on-1 Call | Closing | Decision |

### 1.4 Content Decision If-Then

| Decision Point | Condition | Action |
|---------------|-----------|--------|
| Topic selection | First-hand experience + trend connection | → Start Major Master immediately |
| Topic selection | No first-hand experience + no differentiation | → Reject or change angle |
| Publish decision | Material meets <3 promotion criteria | → Stock it or gather more info |
| Channel selection | Distributing from one master | → Follow channel role map. No copy-paste |
| Quality check | Content sounds AI-generated | → Rewrite immediately |
| Hook judgment | Fewer than 3 hook options | → Not ready. Generate 3+ options and compare |

## 2. Business & Product Decision Knowledge

### 2.1 Core Promise

- **Define your core promise here**: What transformation do you deliver?
- **One-line definition**: Who do you serve + what problem do you solve + how
- **Unique strength**: What makes your approach different from everyone else's?

### 2.2 Audience Segments

Define your segments. Example structure:

**Segment A: Beginners**
- Goal: First sale / first result (smallest success experience)
- CTA: Free resource (low commitment entry point)
- Products: Entry-level → Mid-tier → Premium

**Segment B: Established (Primary focus)**
- Goal: Systems that run without constant manual effort
- Core offer: Audit/diagnosis → Gap identification → Solution
- CTA: Application-based → Call → Premium offer

### 2.3 Business Levers (Strongest first)

1. **Revenue Design** — Price optimization, LTV maximization, offer variety, launch frequency
2. **Acquisition Design** — List building, traffic, funnels, SEO
3. **Automation Design** — Support, education, follow-up automation
4. **Diagnostics** — Bottleneck identification → conversion to sales

### 2.4 Funnel Type Selection

| Condition | Recommended Type |
|-----------|-----------------|
| Price under $1,000 | Direct sale (education → landing page → checkout) |
| Value is best shown through experience | Challenge type (5-day event → enrollment) |
| Requires 1-on-1 consultation | Consultation type (diagnosis → call → close) |
| Other | Event type (webinar → application) |
| Combination | Webinar → 1-on-1 consultation (2-step) |

### 2.5 Pricing Decision Axes

- Beginners can't afford premium → Create stepping stones (entry → mid → premium)
- Established audience → Premium OK (they can visualize ROI)
- Low-price products have low revenue impact → Long-term play. Prioritize premium short-term
- Never put exact prices in content — save for sales conversations

### 2.6 Product Decision If-Then

| Decision Point | Condition | Action |
|---------------|-----------|--------|
| New product vs improve existing | Existing product CVR is low | → Improve existing first |
| New product vs improve existing | Clear gap in market | → New product, but start with MVP |
| Pricing | Target is beginners | → Start under $1,000 |
| Pricing | Target is established + can visualize ROI | → Premium ($5K-$50K+) |
| Short-term vs long-term | No sales system yet | → Short-term (premium) priority |
| Short-term vs long-term | Revenue foundation in place | → Long-term (low-ticket, brand building) |

## 3. Team & Resource Decision Knowledge

### 3.1 Leadership Decision Axes

- **Challenge vs Problem**: Challenge = moving forward (toward goal). Problem = fixing backward (restoring)
- **Communication gap**: "I thought I communicated" vs "Instructions were unclear" — this gap is structural. Always define the ideal end state explicitly
- **Quantity vs Quality trade-off**: Chasing volume can erode quality. Monitor both metrics

### 3.2 Task Management Principles

- Time spent deciding what to do is waste → Clarify goal → Decompose tasks → Execute top-down
- **Single-tasking**: Context-switching kills productivity. Focus on one thing at a time
- **Measure your time**: "I'm working" without measurement is self-deception. Tracking creates awareness
- **Map all tasks → Automate revenue-connected ones first**: You can't automate what you haven't mapped

### 3.3 Design for Low Motivation

- Motivation doesn't last → Small wins + accountability structures
- Goal → Micro-tasks → Lower the barrier to starting
- Reduce friction on tedious tasks (templates, shortcuts, habits)
- **Would this work at 60% motivation?** If not, the system needs more structure

### 3.4 Outsource / Delegate Decision

| Condition | Action |
|-----------|--------|
| You don't need to do it + money solves it | → Outsource. Use freed time for core work |
| Repetitive task + pattern is clear | → Automate (scripts, templates, AI tools) |
| Requires your thinking/judgment | → Do it yourself. Use AI as thinking partner |
| Creative/exploratory work | → Do it yourself. The experience becomes your next product |

### 3.5 Community & Environment

- Your level rises with the level of people around you
- Solo work leads to quitting → Environment provides continuity
- "Boot camp energy" and accountability sustain momentum
- People who are stuck don't need more information — they need an environment
- Accompaniment > Information: People need someone running alongside them, not more knowledge

### 3.6 Resource Allocation If-Then

| Decision Point | Condition | Action |
|---------------|-----------|--------|
| Time allocation | Revenue-direct vs indirect activity | → Revenue-direct ≥ 70% |
| New vs existing | Existing systems incomplete | → Complete existing first. New projects later |
| Automation priority | Repetitive + revenue-connected | → Automate ASAP |
| Investment | Knowledge vs financial | → Knowledge investment compounds faster |
| Skill building | Broad vs deep | → Go deep on one thing first. Expand later |

## 4. Strategy Decision Knowledge

### 4.1 Strategic Principles

- **Structure > Effort**: "Try harder" is banned. Redesign the structure
- **What NOT to do > What to do**: Decide what to cut first. Focus
- **80% is passing**: Perfectionism kills output volume. Ship, then improve what works
- **Market First**: Reverse-engineer from money flow, not vision
- **Top-down design**: Goal → Monthly targets → Controllable metrics

### 4.2 Competitive Analysis (The Real Kind)

- Most people just make a list → That's "pretend analysis"
- **Real competitive analysis**: Experience the competitor's product up to the point of purchase
- For service businesses: Actually go through their funnel, read their emails, attend their webinars
- **Pyramid thinking**: Don't only study the top player (leads to despair). Research realistic peers
- **Concrete → Abstract → Concrete**: Study individual successes → Extract patterns → Apply to yourself
- **A × B combination**: One thing alone = copying. Cross-industry combination = differentiation
- Follower count ≠ revenue. Verify actual business results

### 4.3 Contrarian Thinking

- Going against the crowd often leads to better outcomes
- Convention-breaking content performs well on social
- But: Don't be contrarian for its own sake → Evidence-based contrarian positions only
- Critical thinking: Question premises, find logical contradictions, judge based on evidence

### 4.4 Market Timing Signals

| Signal | Meaning | Action |
|--------|---------|--------|
| New tool/technology appears | Everyone starts from zero → Credentials not needed | → Touch it first, publish first |
| Competitors retreat/weaken | Position opens up | → Claim the differentiation point |
| Demand spike (buzz/news) | Short-term acquisition opportunity | → Emergency content, ship immediately |
| Regulation change (platform policy) | Incumbent advantages break | → Establish optimal strategy under new rules first |
| Same pain point repeated in customer voice | Market pain is crystallizing | → Productize or create content |

### 4.5 Pivot Decision

| Condition | Action |
|-----------|--------|
| 3 months with same tactics, no improvement | → Consider pivot. Question the premise, not the tactics |
| Customer voice diverges from product | → Adapt product to customer voice (never the reverse) |
| Market trend clearly shifted | → Re-run competitive analysis → Strategy review |
| You have a loyal audience | → Genre change is OK. Fans follow the person |
| Stubbornly attached to one thing | → Have the courage to move to the next opportunity |

### 4.6 Strategy Decision If-Then

| Decision Point | Condition | Action |
|---------------|-----------|--------|
| New market entry | Competitors are weak + you have first-hand experience | → GO. Focus on building proof |
| New market entry | Uncharted territory (new tools, etc.) | → GO. Credentials not needed |
| Scale decision | Hitting 1:1 capacity limits | → Shift to content products or team |
| Platform choice | Starting from zero | → Start where the audience already is |
| Information investment | Instructor has results + their students have results | → Investment OK |
| Trend response | 80% latest info + 20% project-based | → This balance for content |

## 5. Calibration (Reference Data for Decision Accuracy)

### 5.1 Common Decision-Making Biases

| Bias | Symptom | Counter |
|------|---------|---------|
| Structure worship | Trying to solve everything with systems | Some problems are emotional/relational — structure won't fix them |
| Efficiency obsession | Cutting everything that looks "wasteful" | Serendipity and exploration produce unexpected value |
| Speed over quality | Rushing to ship | 80% is the floor. 60% is a different (bad) thing |
| Lateral expansion greed | Trying to expand everything at once | Focus first. Expand only after proof of concept |
| Shiny object syndrome | Jumping to new tools/trends | Ask "does this connect to revenue?" first |

### 5.2 Speed vs Quality Matrix

| | Reversible | Irreversible |
|---|---|---|
| **Urgent (today)** | → 80% and ship | → Minimum quality check, then ship |
| **Important, not urgent** | → Aim for 90% | → Research + careful judgment |
| **Neither** | → Defer or cut | → Don't do it |

### 5.3 Uncertainty Protocol

| State | Condition | Action |
|-------|-----------|--------|
| **Decide now** | Reversible + enough info + delay is costly | → Decide. Fix if wrong |
| **Ask** | Irreversible or info gap + someone can answer | → Ask. Don't guess |
| **Wait** | Irreversible + info gap + more info coming | → Intentionally defer. Set a deadline |

### 5.4 ROI Filter

```
ROI Score = (Impact × Success Probability) ÷ Effort

Impact: High(3) / Medium(2) / Low(1)
Success Probability: High(3) / Medium(2) / Low(1)
Effort: Heavy(3) / Medium(2) / Light(1)

Score 3+ → Do it
Score 1-2 → Defer or conditional
Score <1 → Don't do it
```
