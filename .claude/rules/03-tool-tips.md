# ツールの使い方と制限（全会話に自動適用）

Claude Code のツールには制限がある。知っておくと無駄なエラーが減る。

---

## コアツール一覧

| ツール | 用途 |
|--------|------|
| Read / Write / Edit | ファイルの読み書き・編集 |
| Glob | ファイル名パターンで検索（find の代わり） |
| Grep | ファイル内容を検索（grep の代わり） |
| Bash | ターミナルコマンドの実行 |
| WebFetch | 公開URLからコンテンツ取得 |
| WebSearch | Web検索 |
| AskUserQuestion | ユーザーへの質問 |
| Task | サブエージェントの起動 |

---

## 既知の制限

| 制限 | 詳細 | 回避策 |
|------|------|--------|
| 認証付きURL | WebFetch は認証が必要なURL（X, Google Docs等）を取得できない | ユーザーにテキストを貼ってもらう |
| PDF | 1度に最大20ページ | `pages` パラメータで範囲指定（例: "1-10"） |
| MEMORY.md | 200行で切断される | 索引+サブファイル構成にして200行以内に収める |
| スキルの認識 | `.claude/skills/*/SKILL.md` の構造のみ認識 | フォルダ名がスキル名。SKILL.md は大文字必須 |
| 並行Edit | 複数ファイルを同時に Edit すると、1つ失敗で全部失敗 | 失敗時は Read で最新を取得→やり直し |
| 長い出力 | 出力が長すぎると途中で切れる | 分割して出力するよう指示する |

---

## 失敗時の対処パターン

| 失敗 | 対処 |
|------|------|
| Edit失敗（old_string不一致） | Read で最新内容を取得→正確な文字列で再実行 |
| Write後ファイルが見つからない | `ls` でパスを確認→形式を修正して再試行 |
| WebFetch認証エラー | ユーザーにテキスト貼り付けを依頼 |
| Bashタイムアウト | timeout パラメータを延長（最大600000ms） |
| サブエージェントが動かない | prompt に必要な情報が足りない可能性。詳細を追加 |

---

## 使い分けのコツ

- ファイル操作には**専用ツール**を使う（Bash で cat, grep, find しない）
- 単純な検索は Glob/Grep で直接。複雑な調査は Task + Explore エージェント
- 独立した操作は**並列実行**（同時に複数ツールを呼ぶ）
- 依存する操作は**順番に実行**（前の結果が次に必要な場合）
