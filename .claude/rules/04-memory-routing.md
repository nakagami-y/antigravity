# メモリの整理ルール（全会話に自動適用）

Claude Code には auto memory（自動記憶）機能がある。
情報がゴチャゴチャにならないよう、何をどこに保存するかのルールを決めておく。

---

## 保存先の判定フロー

新しい情報を受け取ったとき:

```
新情報
  ↓
1. 一時的な情報か？（今日のタスク完了、等）
   → Yes → 保存しない
  ↓ No
2. 技術的な教訓か？（「○○するとエラーになる」等）
   → Yes → CLAUDE.md の「メモ」セクションに追記
  ↓ No
3. ユーザーの好み・ルールか？（「一人称は○○で」等）
   → Yes → auto memory に保存（「覚えておいて」）
  ↓ No
4. プロジェクトの方針変更か？
   → Yes → CLAUDE.md の該当セクションを更新
  ↓ No
5. 作業の計画 or 結果か？
   → Yes → memory_bank/plan/ or memory_bank/report/ に保存
  ↓ No
6. ルール化すべきか？（「毎回○○してほしい」等）
   → Yes → .claude/rules/ に新しいルールファイルを作成
```

---

## 矛盾の検知

以下の状況に気づいたら、処理を止めてユーザーに確認する:

1. **ルール間の矛盾**: 2つのルールが相反する指示を出している
2. **メモリと現実の不一致**: 記録された情報と実際のファイル状態が食い違う
3. **指示間の矛盾**: 今回の指示と過去の指示が食い違う

確認の仕方: 「○○と△△が矛盾しています。どちらを優先しますか？」と具体的に提示する。推測で解決しない。

---

## メモリの衛生管理

定期的に以下をチェックする（月1回程度）:

- [ ] CLAUDE.md の「メモ」が長くなりすぎていないか（長い場合はルール化 or 分離）
- [ ] memory_bank/plan/ に古い計画が放置されていないか（完了 or 不要なら削除）
- [ ] memory_bank/report/ が増えすぎていないか（古いものはアーカイブ検討）
- [ ] auto memory に矛盾する記録がないか
