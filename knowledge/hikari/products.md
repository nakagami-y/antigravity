# ひかりのAI大学 -- 商品体系

## このファイルについて

ひかりのAI大学の導線設計・商品構成・価格設計・購入トリガーをまとめたドキュメント。
YouTube無料動画からサブスクコミュニティまで、どのように顧客を導くかの全体像を記載している。

---

## 1. 導線設計（B2C基本フロー）

### 全体像

```
YouTube（無料・実演動画）
  ↓ 動画内CTA
公式LINE登録（診断/特典/テンプレ配布）
  ↓ LINE内ナーチャリング
有料ライブセミナー（¥3,000）
  ↓ セミナー内オファー
サブスクコミュニティ（月額¥980・暫定）
```

### 各ステップの役割

| ステップ | 役割 | 提供するもの |
|---------|------|------------|
| YouTube | 「できそう」と思わせる実演 | 無料の実演動画、手順解説、ツール紹介 |
| LINE登録 | 診断で「あなた向け」を提示し初回成功体験を作る | AI業務診断、テンプレート、特典コンテンツ |
| 有料ライブ | 成功体験を深め、サブスクへの信頼を構築 | ライブ実演、質疑応答、個別アドバイス |
| サブスク | 継続的な成功体験とコミュニティ | 動画/記事/テンプレ/チェックリスト、質問・添削・相談・事例共有 |

> **LINE→サブスクの詳細設計**: 商品リリースタイミング（月1セミナー）、サブスク誘導の心理設計、フェーズ1（小さな成功体験）・フェーズ2（プレプレローンチ）は [line-funnel.md](./line-funnel.md) を参照。

### 設計思想

- **CTA（行動喚起）は原則1つ**: 「公式LINE登録」に集約する
- **押し売りしない**: 価値を先に渡し、行動は視聴者/読者に委ねる
- **選択肢を増やさない**: 各段階で「次にやること」を1つに絞る

## 2. B2Bの使い分け

法人向けとB2C向けで、利用するプラットフォームを分けている。

| 用途 | プラットフォーム | 理由 |
|------|----------------|------|
| 法人チャット/コミュニティ | Slack | ビジネス感があり、法人が導入しやすい。Discordは「遊び感」の懸念あり |
| 集客（リスト化） | LINE | B2C集客の中心。診断・特典配布・ナーチャリングに最適 |

## 3. 商品コンセプト（Before → After）

### 提供価値の定義

商品の価値は「知識が増えること」ではなく「状態が変わること」。
以下のフレームで商品を設計する。

| 項目 | 内容 |
|------|------|
| 誰の | AIをやった方がいいと思いながら動けていない非エンジニアのビジネス職 |
| どんな状態を（Before） | AI情報を見ているが実務で使えていない。面倒な作業を手作業でやっている |
| どこまで変えるか（After） | 自分の業務でAIを使い、具体的に仕事が減っている状態 |
| どれくらいで（期間） | 初回成功体験は当日~1週間以内。継続的な改善はサブスクで |
| 成果指標 | 「今日1つ仕事が減った」実感。作業時間の削減。コピペ可能なテンプレが手元にある |

### Before → After の具体例

| Before | After |
|--------|-------|
| Excel集計を毎回手作業でやっている | AIにデータ整理を任せ、確認だけで済む |
| メール返信に30分かかっている | テンプレ+AIで5分に短縮 |
| 議事録を手書きでまとめている | AIで自動要約し、確認・修正だけ |
| AIツールが多すぎて何を使えばいいか分からない | 診断で「あなたはこれ」と1つに絞られている |
| AIを試したが結局続かなかった | クエスト化で小さな成功体験を積み、継続できている |

## 4. Unique Mechanism（なぜこの方法で成果が出るのか）

### 診断 x テンプレ x 実演

ひかりのAI大学の独自メカニズムは、3つの要素の組み合わせで「迷い」を設計で潰すこと。

| 要素 | 役割 | 具体的な仕組み |
|------|------|-------------|
| 診断 | 「何を使うか」の迷いを消す | LINE登録後にAI業務診断。業務内容に応じて最適なツールと手順を1つに絞って提示 |
| テンプレ | 「どうやるか」の迷いを消す | コピペ可能なプロンプト、チェックリスト、手順書を提供。ゼロから考えなくていい |
| 実演 | 「本当にできるか」の迷いを消す | YouTube動画で画面を見せながら実際にやる。「見たまま真似すればできる」状態を作る |

### 競合との違い

- 一般的なAI講座: 「まず理論を学び、次に実践」（学習 → 実行）
- ひかりのAI大学: 「まず成功体験、必要なら後で理解」（実行 → 学習）

**"AIの正解"を短距離で提示する（情報過多の解毒）**

## 6. 価格設計の考え方

### 価格体系

| 商品 | 価格 | 位置付け |
|------|------|---------|
| YouTube動画 | 無料 | 入口・信頼構築 |
| LINE登録 | 無料 | 診断・初回体験 |
| 有料ライブ | ¥5,000 | フロント |
| サブスク | 月額¥3,980（暫定） | バック |

### 価格設計の原則

| 要素 | 考え方 |
|------|--------|
| アンカー（比較対象） | 他のAI講座やスクールの価格と比較して、手が出しやすい価格帯に設定 |
| 分割/年払い | サブスクは月額で心理的ハードルを下げる。年払いオプションも検討 |
| アップセル | 有料ライブ → サブスクへの自然な接続。サブスク内での追加オプション |
| ダウンセル | サブスクが高いと感じる層には、単発ライブで成功体験を提供 |
| 返金/解約条件 | サブスクはいつでも解約可能。解約障壁を下げて信頼を作る |

### 価格設計の完了条件

- 価格の根拠が30秒で説明できる
- 「この条件なら買う（購入トリガー）」が3つ以上特定されている

## 7. 購入トリガー（何があれば買うか）

### 典型的な購入トリガー

| トリガー | 具体例 |
|---------|--------|
| 成功体験の実感 | YouTubeやLINEで実際に仕事が減った体験をした |
| 「自分にもできそう」感 | 実演を見て「これなら真似できる」と感じた |
| 損失回避 | 「手作業で損している時間」を具体的に認識した |
| コミュニティの魅力 | 「一人でやるより、質問できる環境がほしい」 |
| 限定感・タイミング | ライブセミナーの日程が合った、特典がついている |

### 典型的な拒否条件（買わない理由）

| 拒否条件 | 対応方針 |
|---------|---------|
| 「自分の仕事では使えなさそう」 | 診断で業務に当てはめる |
| 「難しそう」 | コピペ/手順/画面付きで「見るだけ」を訴求 |
| 「情報漏洩が怖い」 | NG情報の線引き、代替手段を提示 |
| 「結局続かなかった」 | クエスト化/成功体験/コミュニティのサポートを訴求 |
| 「無料で十分」 | 無料では得られない「個別最適化」「質問環境」を訴求 |

## 8. オンボーディング設計（最重要）

サブスクの成否はオンボーディング（初回体験）で決まる。

### オンボーディングの原則

1. **登録直後に「これをやればOK」を1つ提示する**（選択肢を増やさない）
2. **初回成功体験を当日中に作る**（小さくていい。「AIでメールの下書きを作った」でOK）
3. **次のステップを見せる**（ロードマップ/クエスト形式で「次はこれ」を提示）
4. **質問できる場所を明示する**（「困ったらここで聞いてOK」）
5. **放置しない**（1週間以内にフォローアップ）

### 初回成功体験の例

| 業務 | 初回成功体験 | 所要時間 |
|------|------------|---------|
| 事務/総務 | AIでメールの定型文を自動生成した | 10分 |
| 営業 | AIで提案書のたたき台を作った | 15分 |
| 経理 | AIでExcelデータの集計ルールを作った | 20分 |
| 管理職 | AIで議事録の要約を作った | 5分 |
