# ひかりのAI大学 -- ターゲット・市場分析

## このファイルについて

ひかりのAI大学のターゲット定義、市場ポジション、顧客理解のフレームワーク、競合分析テンプレート、需要検証方法をまとめたドキュメント。
「誰に、何を、なぜ」を設計・検証するための基盤資料。

---

## 1. ターゲット定義

### コアセグメント

> **AIをやった方がいいと思いながら動けていない人（ライト実務層）**

「AIに興味がない人」ではなく、「興味はあるが行動に移せていない人」が中心。
原因は「知らない」ではなく「動けない」にある。

### セグメント設計の原則

ペルソナ（架空の1人を描く手法）ではなく、**セグメント（属性 x 状況 x 欲求）**で設計する。
1人の理想像ではなく、共通の属性・状況・欲求を持つグループとして捉える。

### セグメントの3軸

| 軸 | 内容 | 具体例 |
|----|------|--------|
| 属性 | 職種・役職・業種 | 非エンジニアのビジネス職（事務/総務/経理/営業/企画/管理職など） |
| 状況 | 今どういう状態にあるか | YouTubeやSNSでAI情報を見るが、実務では使っていない |
| 欲求 | 本音で何を求めているか | 「AIを学びたい」ではなく「楽をしたい」「仕事を減らしたい」「損したくない」「置いていかれたくない」 |

### ターゲットの本音（欲求の優先順位）

1. **楽をしたい** -- 面倒な作業を減らしたい。手を抜きたいわけではなく、もっと効率よくやりたい
2. **仕事を減らしたい** -- 残業を減らしたい。早く帰りたい。本来やるべき仕事に集中したい
3. **損したくない** -- 手作業で時間を浪費していることに気づきたい。もったいないことをしたくない
4. **置いていかれたくない** -- 周りがAIを使い始めている。自分だけ取り残されたくない

**注意**: 「AIを学びたい」はターゲットの本音ではない。手段（AI学習）を目的にしない。

## 2. 実行摩擦4パターン

ターゲットが「やった方がいいと思いながら動けない」理由を4つに分類する。
コンテンツ設計・反論処理・CTA設計のすべてで、この4パターンへの対処を意識する。

### 4パターンの詳細

| # | パターン | 内面の声 | 典型的な行動 | 対処方針 |
|---|---------|---------|------------|---------|
| 1 | 翻訳コスト | 「AIの話は分かるけど、自分の仕事にどう関係するの？」 | AI動画を見るが、自分の業務への適用方法が分からず止まる | 業務別の具体例を提示。診断で「あなたの業務なら○○」と当てはめる |
| 2 | 選択疲れ | 「ChatGPT? Copilot? Gemini? どれを使えばいいの？」 | ツール比較記事を読み漁るが、結局どれも試さない | 「おすすめは1つ」に絞る。選択肢を増やさず、迷いを消す |
| 3 | 心理コスト | 「情報漏洩したらどうしよう」「会社で怒られない？」 | 興味はあるが、リスクが怖くて手を出せない | NG情報の線引きを明示。安全な使い方を具体的に提示 |
| 4 | 動機の弱さ | 「別にそこまでやる気はない。できれば楽したいだけ」 | AI情報を見て「いいな」と思うが、わざわざ時間を使って取り組まない | 「10分でできる」「コピペだけ」など、最小コストで始められることを強調 |

### 実行摩擦への対処がコンテンツの核

ひかりのAI大学のコンテンツは、すべてこの4パターンのどれかを解消するために作る。

| コンテンツの種類 | 主に対処する摩擦 |
|---------------|---------------|
| YouTube実演動画 | 翻訳コスト（「自分の仕事で使える」を見せる） |
| LINE診断 | 選択疲れ（「あなたはこれ」を1つに絞る） |
| テンプレ/コピペ素材 | 動機の弱さ（「コピペだけ」で始められる） |
| FAQ/不安解消コンテンツ | 心理コスト（「これなら安全」を示す） |

## 3. 市場ポジション

### AI教育市場ではなく、ライト自己改善市場

ひかりのAI大学が戦う市場は「AI教育」ではない。

| 市場 | 競合例 | 顧客の期待 | ひかりのAI大学の立ち位置 |
|------|--------|-----------|---------------------|
| AI教育市場 | AI系スクール、Udemy、CourseraのAIコース | 「AIを体系的に理解したい」 | ここでは戦わない |
| ライト自己改善市場 | 仕事術系YouTube、時短系メディア、ライフハック系 | 「楽をしたい」「効率化したい」「損したくない」 | ここで戦う |

### ポジションの差別化

「AI教育のポジションを取りに行くのではなく、仕事術/時短市場にAIという手段を持ち込む」

| 差別化の軸 | 一般的なAI教育 | ひかりのAI大学 |
|-----------|-------------|-------------|
| 提供する価値 | 知識・理論・体系 | 時短・楽・仕事が減る実感 |
| 成果の測定 | 理解度（テスト等） | 実際に減った作業時間 |
| 学習の順序 | 理論 → 実践 | 実践（成功体験） → 必要に応じて理解 |
| コンテンツの核 | 講義・解説 | 実演・テンプレ・診断 |
| 差別化の芯 | カリキュラムの質 | 「ここを見ればAIの正解がわかる」権威性 |

## 4. 顧客理解のフレームワーク

新しい商品やコンテンツを設計する前に、以下のフレームワークでターゲットの現状を整理する。

### 4-1. 顧客の現状を埋める（最低限の4問い）

| 問い | 答えるべき内容 | 記入例 |
|------|-------------|--------|
| 苦痛（Pain） | 何が面倒？何に時間が溶ける？ | 「メール返信に毎日30分。議事録まとめに1時間」 |
| 代替案（Alternative） | 今はどうしている？ | 「Excel手作業。テンプレ探し。外注」 |
| 失敗経験（Failure） | AIで何を試して何がダメだった？ | 「ChatGPTに聞いたが的外れな回答だった」 |
| 購入障壁（Barrier） | 何が不安？ | 「価格。難しさ。社内規定。効果が不明」 |

### 4-2. 意思決定要因

| 項目 | 内容 |
|------|------|
| 購入トリガー | 何があれば買う？（成功体験の実感、具体的な数字、コミュニティの存在、期間限定等） |
| 拒否条件 | 何があると買わない？（難しそう、高い、効果不明、押し売り感等） |
| 反論処理 | 買わない理由への反論を10個以上用意する |

## 5. 競合棚卸しテンプレート

新しい商品・キャンペーン・コンテンツシリーズを開始する前に、競合を棚卸しする。

### フォーマット

| 比較軸 | 競合A | 競合B | 競合C | 競合D | 競合E | ひかりのAI大学 |
|--------|-------|-------|-------|-------|-------|-------------|
| 価格 | | | | | | |
| 形態（動画/テキスト/ライブ等） | | | | | | |
| 期間（単発/サブスク/買い切り） | | | | | | |
| サポート（質問/添削/コミュニティ） | | | | | | |
| 証拠（実績/事例/数字） | | | | | | |

### 棚卸しのルール

- **競合は5つ**: 多すぎると分析が散漫になる。5つに絞る
- **比較軸は5つ**: 価格/形態/期間/サポート/証拠を基本とし、必要に応じて追加
- **差別化余地は1つに集約**: 「あれもこれも違う」ではなく、最大の差別化ポイントを1つに絞る
- **差別化をメッセージ案に落とす**: 棚卸しだけで終わらず、訴求に使えるメッセージに変換する

### 競合の範囲

直接競合（AI教育系）だけでなく、間接競合（仕事術系、時短系）も含める。
ターゲットの「時間と注意の競合相手」として捉える。

| 競合の種類 | 例 |
|-----------|---|
| 直接競合（AI教育） | AI系YouTuber、AI系オンラインスクール |
| 間接競合（仕事術） | 仕事術系YouTuber、時短系メディア、ライフハック系コンテンツ |
| 代替手段 | 「AIを使わない」選択肢（手作業の継続、外注、既存ツール） |

## 6. 需要検証方法

商品やコンテンツのアイデアを実行に移す前に、需要があるかを検証する。

### 検証方法（推奨順）

| # | 方法 | 内容 | 信頼度 |
|---|------|------|--------|
| 1 | プリセール/先行モニター募集 | 実際にお金を払う人がいるかを確認。最も信頼度が高い | 高 |
| 2 | 事前登録/ウェイトリスト | LP等で「興味がある人」の数を計測する | 中 |
| 3 | 簡易LP + 広告/投稿 | LPを作り、広告やSNS投稿でトラフィックを流してCTR・CVRを見る | 中 |
| 4 | SNSの反応 | 投稿への反応（いいね/リプ/保存）で需要の温度感を測る | 低 |

### 検証の判断基準

| 指標 | 基準（目安） |
|------|------------|
| ウェイトリスト登録率 | LP訪問者の10%以上で「需要あり」 |
| プリセールの成約 | 3人以上が購入したら「作っていい」 |
| X投稿の保存率 | 保存 / インプレッション が1%以上で「テーマに需要あり」 |

### 検証前にやること

- ターゲットセグメントを明確にする（4-1の4問いを埋める）
- 提供価値のBefore→Afterを1文で書ける状態にする
- 競合棚卸しで差別化ポイントを1つ確認する

## 7. セグメント別アプローチ例

コアセグメント内でも、属性によってアプローチの切り口を変える。

| 属性 | 刺さる切り口 | コンテンツ例 |
|------|------------|------------|
| 事務/総務 | 「Excel集計、毎月やってません？」 | AIでExcel作業を時短する実演動画 |
| 営業 | 「提案書のたたき台、30分かけてません？」 | AIで提案書を5分で作る手順 |
| 経理 | 「月次の集計、手作業ですか？」 | AIで経理業務を自動化するテンプレ |
| 管理職 | 「議事録、部下に書かせてません？」 | AIで議事録を自動要約する方法 |
| 企画 | 「リサーチに半日使ってません？」 | AIで市場調査を1時間に圧縮する手順 |

### アプローチの原則

- **職種別の「痛み」から入る**: 「AI便利です」ではなく「○○の作業、面倒ですよね？」から
- **手順は具体的に**: 「AIを使えば楽になります」ではなく「この3ステップでOK」
- **おすすめは1つに**: ツール比較ではなく「あなたの業務なら○○一択」

## 8. 顧客理解の更新ルール

顧客理解は固定ではなく、以下のタイミングで更新する。

| タイミング | 更新内容 |
|-----------|---------|
| LINE/コメントで質問が来たとき | 「苦痛（Pain）」「購入障壁（Barrier）」に追加 |
| 新しい事例が出たとき | Before→Afterの具体例として追加 |
| 解約/離脱が発生したとき | 拒否条件・失敗経験に追加して対策を検討 |
| 新商品を企画するとき | 4問い + 競合棚卸し + 需要検証を最初から回す |
