## 2. コンセプト

### 1文コンセプト

> **考えなくても、選ばなくても、ここだけ見ればAIを使いこなせる（仕事が減る）。**

### コンセプトの背景

AI情報は無料・有料ともに十分な量がある。それでも「使えていない人」が多い。
原因は「知らない」ではなく、**行動できていない（実行摩擦）**にある。

実行摩擦は主に4つのパターンで発生する:

| パターン | 具体的な状態 |
|---------|------------|
| 翻訳コスト | 自分の業務にどう関係するか分からない |
| 選択疲れ | ツールが多すぎて選べない |
| 心理コスト | 失敗・セキュリティ・社内規定が怖い |
| 動機の弱さ | 仕事を追求するほどのやる気はない。できれば楽したい |

この4つの摩擦を「設計で潰す」のが、ひかりのAI大学のアプローチ。

## 3. 3つの「しなくていい」（提供価値の柱）

ひかりのAI大学が約束する価値は、**知識を増やすこと**ではなく**迷いと手間を減らすこと**。
これを3つの「しなくていい」で実現する。

### 3-1. 覚えなくていい

- テンプレート、自動化ツール、コピペ資産を提供する
- 記憶に頼らず、コピペで再現できる状態を作る
- 「覚える」より「使えるものが手元にある」を重視

### 3-2. 選ばなくていい

- 診断で「あなたにはこれ」とおすすめツール・手順を1つに固定する
- 「選択肢を増やす」のではなく「迷いを消す」
- 情報過多の解毒として機能する

### 3-3. 学ばなくていい

- 体系学習より「今日ラクになる手順」を先に渡す
- 理論を学んでから実践するのではなく、まず実践して結果を出す
- 学習の順序を逆転させ、成功体験から入る

### 補足: 継続の仕組み

- ゲーミフィケーション（学習/実行ロードマップ、クエスト化）で成功体験を積み重ね、継続モチベーションを維持する

## 4. 市場ポジション

### ポジショニングの考え方

ひかりのAI大学は「AI教育市場（知識・理論）」ではなく、**ライト自己改善（仕事術/時短/不安解消）市場**に位置する。

| 比較軸 | AI教育市場（一般的なAI講座） | ひかりのAI大学のポジション |
|--------|-------------------------|------------------------|
| 主な提供物 | 知識、理論、体系的カリキュラム | 手順、テンプレ、診断、実演 |
| 読者の期待 | 「AIを理解したい」 | 「仕事を減らしたい」「楽したい」 |
| 成果の定義 | 知識が増えた | 仕事が実際に減った |
| 競合 | AI系スクール、Udemy等 | 仕事術系YouTube、時短系メディア |
| 差別化の芯 | -- | 「ここを見ればAIの正解がわかる」権威性 |

### 差別化のキーワード

**「情報過多の解毒」**

- AIツール・情報が溢れている状態で、「結局何を使えばいいの？」に答える存在
- 選択肢を狭めることが価値になる（他のメディアが選択肢を広げる中で逆張り）

## 5. ターゲット像

### コアセグメント

> **AIをやった方がいいと思いながら動けていない人（ライト実務層）**

| 属性 | 内容 |
|------|------|
| 職種 | 非エンジニアのビジネス職（事務/総務/経理/営業/企画/管理職など） |
| 状況 | YouTubeやSNSでAI情報を見るが、実務で使っていない |
| 本音の欲求 | 「AIを学びたい」ではなく「楽をしたい」「仕事を減らしたい」「損したくない」「置いていかれたくない」 |

### ターゲットが持つ典型的な不安

1. 「自分の仕事では使えなさそう」（翻訳コスト）
2. 「難しそう」（技術的ハードル）
3. 「情報漏洩が怖い」（セキュリティ不安）
4. 「結局続かなかった」（過去の失敗経験）

## 6. 中長期戦略

### 差別化の芯

「ここを見ればAIの正解がわかる」という**権威性の確立**。

### 中長期の方向性

| 施策 | 狙い |
|------|------|
| 自社学習プラットフォーム（アプリ）構想 | 他プラットフォーム依存からの脱却 |
| ゲーミフィケーション | 学習/実行ロードマップのクエスト化で継続率を上げる |
| コミュニティ（サブスク） | 成功体験の共有と相互サポートで離脱を防ぐ |

## 7. AIへの指示原則（コンテンツ制作時の前提ルール）

ひかりのAI大学のコンテンツをAIで制作する際、以下のルールが適用される。

### 最重要原則

1. **ターゲットは「AIをやった方がいいと思いながら動けていない人」**
2. **価値は「知識」ではなく「楽できる実感」= 仕事が減る体験**
3. **台本がある回は、台本を一次情報（正）として横展開する** -- 結論/主張は変えない
4. **CTA（最終アクション）は原則「公式LINE登録」**
5. **誇大表現・効果保証はしない**

### 出力の基本品質

- 読者/視聴者が**今日1つ実行できる**具体性（コピペ可/手順/テンプレ）
- 「選択肢を増やさない」: 迷いを減らす（おすすめを1つに寄せる）
- 結論ファースト、短文、手順は箇条書き
- "知らない"前提で噛み砕くが、幼稚にはしない

### コンプライアンス最小ルール

- 効果保証・断定・誇大表現はしない
- 禁止表現例: 「絶対」「100%」「必ず成功」「誰でも確実に」「放置で稼げる」
- 実績がないことを「ある」と言わない（推測は推測と明記）
