# ひかりのAI大学 -- コンテンツ戦略

## このファイルについて

ひかりのAI大学のコンテンツ制作・横展開の全体設計をまとめたドキュメント。
勝ちパターン、各媒体の役割分担、台本ベース横展開SOP、台本なし制作SOP、入力YAML仕様、チェックリスト、定型文テンプレートを網羅している。

---

## 1. 勝ちパターン（理論より実演）

### 基本方針

ひかりのAI大学のコンテンツは「講義」ではなく「実演」で勝つ。

| やること | やらないこと |
|---------|------------|
| 画面を見せて実際にやる | 理論を長く説明する |
| コピペ可能なテンプレを渡す | 「考え方」だけ伝えて終わる |
| おすすめを1つに絞る | 選択肢を増やす比較コンテンツ |
| 「今日やる1手」を明確にする | 「いつかやりましょう」で締める |

### 3つの勝ちルール

1. **「理論」より実演**: 画面/Excel/テンプレを見せる。口で説明するだけでなく、実際の操作画面を見せる
2. **「見ただけで終わり」を防ぐ**: その場でコピペできる素材を提供する。動画を見た直後に行動できる状態を作る
3. **CTAは1つ**: 行動の選択肢を増やさない。「次にやること」はLINE登録に集約する

## 2. コンテンツの役割分担

各媒体に明確な役割を持たせ、重複ではなく補完関係を作る。

### 媒体別の役割

| 媒体 | 役割 | 特性 | CTA |
|------|------|------|-----|
| YouTube | メイン媒体。実演で「できそう」を作る | 動画で操作画面を見せられる。信頼構築に最適 | 公式LINE登録 |
| 記事（ブログ/Note） | 手順を整理して保存性を高める。SEO流入 | テキストで手順を保存できる。検索/共有に最適 | 公式LINE登録 |
| X（旧Twitter） | 短い成功体験の入口。気づき → 実行 | 拡散力がある。1投稿1メッセージ | 公式LINE登録 |
| LINE | 診断/特典 → 初回成功体験 → 商品案内 | 1対1感。ナーチャリングに最適 | 有料ライブ → サブスク |

> **詳細**: 公式LINEの目的（サブスク誘導）、フェーズ1・2の設計、サブスク誘導の心理（バンドワゴン・接触頻度・社会的証明）は [line-funnel.md](./line-funnel.md) を参照。

### 媒体間の連携フロー

```
YouTube動画（実演）
  ↓ 台本を一次情報として
記事化（保存版・手順整理）
  ↓ 記事の要点を抽出して
X投稿（拡散・導線）
  ↓ LINEへ誘導
LINE（診断・特典・ナーチャリング → 商品）
```

## 3. 台本ベース横展開SOP（最頻出フロー）

YouTube台本がある回は、台本を一次情報（正）として各媒体へ横展開する。
これが最も頻繁に使うワークフロー。

### 大原則

- **台本 = 一次情報（正）**
- **結論/主張は変えない**
- **追加していいのは「補足」「例」「注意点」「反論処理」「手順の整形」だけ**

### Step 1: 台本 → 要点マップを作る（必須）

台本を各媒体へ変換する前に、まず以下の7要素を抽出する。

| 要素 | 抽出する内容 | 例 |
|------|------------|---|
| Hook | 冒頭の掴み | 「メール返信に毎日30分、無駄にしてません？」 |
| Promise | 結論・得られる成果 | 「今日から5分で終わるようになります」 |
| Steps | 手順（3~7個） | 「1. ChatGPTを開く 2. テンプレを貼る 3. 出力をコピペ」 |
| Example | 具体例（1~2個） | 「営業メールの返信例」「お礼メールの例」 |
| Pitfalls | つまずき・注意点 | 「個人情報は入れない」「敬語の調整は自分で」 |
| Objections | 想定反論（3~10個） | 「でも情報漏洩が...」→ 回答 |
| CTA | 最終アクション | 「公式LINE登録で診断+テンプレ配布」 |

### Step 2: 要点マップ → 記事（保存版）

要点マップから記事を構成する。

**記事の型**

```
H1: 結論（仕事が減るベネフィット）
  例: 「メール返信を5分に短縮する方法【AIテンプレ付き】」

H2: なぜそれで減るのか（超短く）
  例: 「テンプレを貼るだけで、AIが文面を生成してくれる」

H2: 手順（ステップ形式）
  例: 「Step 1: ChatGPTを開く → Step 2: テンプレを貼る → Step 3: 出力をコピペ」

H2: よくある詰まりポイント
  例: 「個人情報は入れない」「敬語レベルの調整は手動で」

H2: テンプレ/コピペ素材
  例: 実際に使えるプロンプトテンプレートを掲載

H2: まとめ（今日やる1手）
  例: 「今日のメール返信1通を、このテンプレでやってみてください」

CTA: LINE登録
  例: 「もっと時短テンプレが欲しい方は公式LINEで配布中（無料）」
```

### Step 3: 記事/台本 → X投稿（拡散&導線）

**投稿構成**

| 種類 | 本数 | 内容 |
|------|------|------|
| 単発投稿 | 7本 | 学び6本 + 軽プロモ1本 |
| スレッド | 1~2本 | 手順型（Step形式）or 失敗回避型 |

**X投稿のルール**

- 1投稿 = 1メッセージ。複数の学びを1投稿に詰め込まない
- 「選択肢を増やさない」: おすすめは1つに寄せる
- CTAは基本LINE（押し売りしない）
- 軽プロモは7本中1本まで

### Step 4: 台本/記事 → プロモ素材（ローンチ用）

有料ライブやサブスクのプロモーション時に使う素材を生成する。

| タイミング | 内容 |
|-----------|------|
| 事前告知 | 不安 → 解決 → 参加理由の順で構成 |
| 直前 | 締切/損失回避を訴求 |
| 当日 | 参加/アーカイブの案内 |
| 事後 | よくある反論 → 証拠 → 次の行動 |

## 4. 台本なし制作SOP（代替ルート）

YouTube台本がない場合のワークフロー。

### 原則

台本なしの場合も、いきなりX投稿やプロモ素材を作らない。
**まず「一次情報」を作り、そこから横展開する。**

### 手順

```
Step 1: 「記事」か「台本（YouTube構成）」を先に作る
  → これが一次情報になる
  → 入力YAMLを埋めて、記事作成プロンプトで生成

Step 2: 一次情報から横展開
  → 記事 → X投稿
  → 台本 → 記事 → X投稿
  （台本ベースSOPのStep 2以降と同じ流れ）
```

### なぜ一次情報を先に作るのか

- X投稿やプロモ素材だけでは、主張の一貫性が担保できない
- 記事を先に作ることで、結論・手順・注意点が整理される
- 整理された情報からX投稿に変換する方が品質が高い

## 5. 入力YAML仕様

コンテンツ生成時にAIへ渡す入力フォーマット。
このYAMLを埋めてAIに渡すと、ブランドのルールに沿った生成ができる。

### YAML全文

```yaml
project:
  asset_type: "article | x_posts | promo_pack | repurpose_from_script"
    # article: 記事を新規作成
    # x_posts: X投稿を量産
    # promo_pack: プロモ素材一括生成
    # repurpose_from_script: 台本から横展開（最頻出）
  base_material: "script | article | none"
    # script: YouTube台本がある（一次情報として使用）
    # article: 既存記事がある
    # none: ゼロから作る
  goal: "lead | engagement | conversion"
    # lead: LINE登録を獲得
    # engagement: 反応/拡散を獲得
    # conversion: 有料商品への誘導
  primary_cta: "公式LINE登録"
    # 原則固定

audience:
  segment: "AIをやった方がいいと思いながら動けていないライト実務層"
  job_context: "例: 事務/営業/管理職"
  pains:
    - "例: Excel集計が面倒"
    - "例: メール返信が多い"
  anxieties:
    - "例: 情報漏洩が怖い"

content:
  topic: "例: 議事録をAIで秒速化"
  key_takeaways:
    - "今日からやる1手"
    - "失敗しない注意点"
  tools_mentioned: ["ChatGPT", "Copilot", "Gemini"]

offer:
  front: "有料ライブセミナー（例: ¥5,000）"
  back: "サブスク（例: 月額¥3,980）"

proof:
  proof_points:
    - "事例/数値/プロセス（ある範囲で）"

constraints:
  tone: "伴走/ナビ。結論ファースト。短文。具体例多め。"
  forbidden_words: ["絶対", "100%", "必ず", "誰でも確実に"]

output:
  length: "short | medium | long"
    # short: X投稿、LINE短文
    # medium: 記事（2,000~3,000字）
    # long: LP骨子、長尺記事（4,000字+）
```

### 最小入力（必須項目のみ）

```yaml
project:
  asset_type: "repurpose_from_script"
  base_material: "script"
  goal: "lead"
  primary_cta: "公式LINE登録"
audience:
  segment: "AIをやった方がいいと思いながら動けていないライト実務層"
  job_context: "例: 事務/営業"
content:
  topic: "（この台本のテーマ）"
offer:
  front: "有料ライブセミナー（例: ¥5,000）"
  back: "サブスク（例: 月額¥3,980）"
constraints:
  forbidden_words: ["絶対", "100%", "必ず", "誰でも確実に"]
```

## 6. チェックリスト（納品前の品質確認・5項目）

すべてのコンテンツは、納品前に以下の5項目をチェックする。

### チェックリスト

| # | カテゴリ | チェック内容 | 合格基準 |
|---|---------|------------|---------|
| 1 | ターゲット適合 | 「学びたい」ではなく「楽したい」に刺さっているか | 具体的な業務シーンが含まれている。おすすめが1つに絞られている |
| 2 | 実行性 | 今日やる1手が明確か | 手順が3~7ステップで書かれている。コピペ可能なテンプレがある |
| 3 | 反論処理 | 不安/疑念の先回りがあるか | プロモ時は10個以上の反論処理。通常コンテンツは1~3個 |
| 4 | CTA | CTAが1つに絞れているか | LINE登録に集約。押し売り感がない（価値 → 行動の順） |
| 5 | コンプラ | 効果保証/断定/誇大表現がないか | 禁止表現リストに該当する表現がゼロ。実績のない主張がない |

### チェックの運用

- 自分で生成したコンテンツは、投稿/納品前に必ず5項目を通す
- 1つでもNGがあれば、該当箇所を修正してから投稿/納品
- チェック結果は記録しなくてよいが、「なぜOK/NGか」を判断できる状態にする

## 7. 生成タスク別プロンプト

### 7-1. 記事作成プロンプト

**入力**: YAML + （あれば）YouTube台本/箇条書き
**出力**: タイトル案3つ / 構成 / 本文 / まとめ / CTA

**プロンプト要旨**:
- 「ひかりのAI大学」の編集者として、ターゲット（AIをやった方がいいと思いながら動けていないライト実務層）に向けて記事を書く
- 読者が今日1つ実行できるように、手順・テンプレ・注意点を必ず入れる
- 誇大表現は禁止
- 最後は公式LINE登録へ自然に誘導
- 台本がある場合は、結論と手順を改変しない

### 7-2. 記事 → X投稿プロンプト

**入力**: 記事本文
**出力**: 単発7本 + スレ1本（必要なら2本）+ 軽プロモ混ぜ

**プロンプト要旨**:
- 記事の要点を抽出し、X投稿に変換
- 単発7本: 学び6本 + 軽プロモ1本
- スレ: 手順型（Step形式）で1本
- 1投稿 = 1メッセージ。迷いを増やさない
- CTAは公式LINE（押し売り禁止）

### 7-3. プロモ素材一括生成プロンプト

**入力**: YAML（offerを埋める）+ （あれば）台本/記事
**出力**: X投稿/スレ/LINE配信/LP骨子/FAQ/固定コメ

**プロンプト要旨**:
- フロント（有料ライブ）→ バック（サブスク）へ自然に接続するプロモ素材を一括生成
- 反論処理を厚く（10個以上の想定不安 → 回答）
- 証拠は「プロセス」「具体例」「注意点」で補強
- 誇大表現なし
- CTAは公式LINE

### 7-4. 競合棚卸しプロンプト

**入力**: 比較したい競合のリスト（5つ）
**出力**: 比較表 + 差別化余地 + メッセージ案

**プロンプト要旨**:
- 競合5つ、比較軸（価格/形態/期間/サポート/証拠）で表にする
- 差別化余地は1つに集約し、メッセージ案に落とす

## 8. 定型文テンプレート

### 8-1. 30秒説明（エレベーターピッチ）

> 「AIって情報は多いのに、結局使えない人が多いですよね。
> うちは"学習"じゃなくて、あなたの業務に合わせて診断で使うAIを決めて、コピペできる時短レシピを渡すので、考えずに仕事が減ります。
> まずは公式LINEで診断 → 今日1つ成功体験、そこから継続できる仕組みまで用意しています。」

### 8-2. X用: 価値提供 → CTAの定型

> 「今日から1つ仕事を減らすなら、これだけ。
> (1) [手順1] (2) [手順2] (3) [手順3]
> 迷う人はLINEで"あなたの業務"に合わせておすすめ出します（無料）」

## 9. 横展開の最小手順（超短縮版）

忙しいときはこれだけ覚えておく。

```
1) 台本を貼る（なければ記事を先に作る）
2) 要点マップ（Hook/結論/手順/注意/CTA）を抽出
3) 記事化（保存版）
4) 記事からX量産
5) ローンチ期だけプロモ素材も作る
```

## 10. ナレッジ強化の更新ルール

コンテンツ戦略は固定ではなく、以下のサイクルで更新する。

| タイミング | 更新内容 |
|-----------|---------|
| 実際に来た質問（LINE/コメント） | 反論処理の材料として追加 |
| 実例（Before→After）が出たとき | 「証拠」に追記 |
| 伸びた投稿/動画のフック | 勝ちフック辞書に蓄積 |
| 台本テンプレの固定化 | 横展開の速度が上がるため、効果的な型を標準化する |
| 新しいAIツール登場時 | tools_mentionedを更新。既存手順の有効性を再検証 |
