# ひかりのAI大学 -- メッセージ設計（ブランドガイド）

## このファイルについて

ひかりのAI大学のメッセージ設計を1枚にまとめたドキュメント。
Big Idea、訴求軸、反論処理、証拠の棚卸し、トーン&ボイス、禁止表現を網羅している。
YouTubeタイトル/サムネ/台本、記事の導入、X投稿、LP、LINE配信の訴求を一貫させるための基盤。

---

## 1. Big Idea（市場の常識の再定義）

### 市場の常識

「AIは勉強して理解しないと使えない」

### ひかりのAI大学の再定義

> **AIは勉強じゃない。迷いを消して、手順を渡せば仕事は減る。**

### Big Ideaの分解

| 要素 | 意味 |
|------|------|
| 「勉強じゃない」 | AI教育市場（体系的な知識・理論）との差別化。学ぶことが目的ではない |
| 「迷いを消して」 | 選択肢が多すぎて動けない状態を、診断・絞り込みで解消する |
| 「手順を渡せば」 | コピペ可能なテンプレ・ステップ形式の手順を提供する |
| 「仕事は減る」 | 最終的な成果は「知識が増えた」ではなく「仕事が実際に減った」 |

### このBig Ideaを使う場面

- YouTubeのタイトル・サムネイル: 「勉強しなくていい」「手順だけ」系の切り口
- 記事の導入: 「AIを学ぶ必要はありません」から入る
- X投稿: 常識の逆張りとして「勉強するな」を使う
- LP/セールス: 競合との違いを1文で伝える

## 2. 訴求軸（4つの使い分け）

ターゲット（AIをやった方がいいと思いながら動けていない人）に刺さる訴求軸は4つ。
コンテンツの目的やターゲットの温度感に応じて使い分ける。

### 訴求軸一覧

| # | 訴求軸 | 刺さるメッセージ例 | 使いどころ |
|---|--------|-----------------|-----------|
| 1 | 成果（仕事が減る） | 「議事録が5分で終わる」「メール返信が半分に」 | YouTube動画タイトル、記事見出し、LPのヘッドライン |
| 2 | 時間短縮（○分削減） | 「毎日30分の手作業、AIなら3分」 | X投稿（数字で具体化）、記事の手順セクション |
| 3 | 不安解消（失敗しない） | 「情報漏洩しない使い方」「社内規定にも対応」 | 反論処理パート、FAQ、LINE配信 |
| 4 | 損失回避（置いていかれない） | 「手作業の時間、もったいなくないですか」 | プロモ投稿、締切訴求、ライブ告知 |

### 訴求軸の優先順位

1. **成果訴求が最優先**: 具体的な「仕事が減る」実感を伝える
2. **時間短縮は数字で**: 「○分 → ○分」のBefore→Afterが最も説得力がある
3. **不安解消は先回り**: 「でも○○が心配ですよね」と読者の不安を言語化してから解消する
4. **損失回避は控えめに**: 恐怖訴求になりすぎないよう、事実ベースで伝える

## 3. 反論処理（想定反論と回答）

### 主要な反論と処理

ターゲットが「買わない理由」「行動しない理由」として持ちやすい反論を、事前に潰す。

| # | 想定反論 | 回答の方向性 |
|---|---------|------------|
| 1 | 「自分の仕事では使えなさそう」 | 診断で「あなたの業務」に当てはめる。事務/営業/経理など職種別の事例を提示 |
| 2 | 「難しそう」 | コピペ/手順/画面付きで「見るだけでOK」を訴求。技術知識不要であることを強調 |
| 3 | 「情報漏洩が怖い」 | 入れてはいけない情報の線引きを明示。代替手段（ローカルAI等）も紹介 |
| 4 | 「結局続かなかった」 | クエスト化/成功体験の積み上げ/コミュニティのサポートで継続を仕組み化 |
| 5 | 「無料で十分では？」 | 無料で得られるのは「情報」、有料で得られるのは「個別最適化と質問環境」 |
| 6 | 「時間がない」 | だからこそAIで時短。最初の10分だけ投資すれば、毎日30分浮く |
| 7 | 「AI使えなくても困ってない」 | 気づいていないだけで、手作業に時間が溶けている。具体例を提示 |
| 8 | 「どのAIツールがいいか分からない」 | 選ばなくていい。診断で「あなたはこれ」を1つに絞る |
| 9 | 「上司/会社が許可しない」 | 社内規定に配慮した使い方を提示。許可を得やすい提案フォーマットも用意 |
| 10 | 「有料ライブに5,000円は高い」 | 毎日30分の手作業削減 = 月10時間。時給換算でペイする計算を提示 |

### 反論処理の原則

- **先回り**: 読者が「でも...」と思う前に、こちらから言及する
- **具体的に**: 「大丈夫です」ではなく、具体的な手段・数字・事例で回答する
- **押し売りしない**: 反論を潰した後も、決定は読者に委ねる

## 4. 証拠の棚卸し

メッセージの説得力を高めるために、以下の5種類の証拠を蓄積・活用する。

### 証拠の種類と使い方

| # | 証拠の種類 | 内容 | 使いどころ |
|---|-----------|------|-----------|
| 1 | 実績（数字/件数/期間） | チャンネル登録者数、ライブ参加者数、受講者数 | LP、プロモ投稿、信頼構築パート |
| 2 | 事例（Before→After） | 具体的な受講者の変化。「○○分かかっていた作業が○○分に」 | 記事、YouTube動画内、セールスページ |
| 3 | データ（時間削減、作業量削減） | 定量的な数字。「平均○○%の時間削減」 | X投稿、LPの数字セクション |
| 4 | プロセス（再現手順） | 「この手順をやればこの結果になる」という再現性の証明 | YouTube実演、記事の手順パート |
| 5 | 第三者（引用/レビュー/企業導入） | 受講者レビュー、企業での導入事例、外部メディアでの言及 | LP、プロモ、信頼構築 |

### 証拠活用の注意

- 実績がないことを「ある」と言わない（推測は推測と明記）
- 数字は出どころを明確にする
- 事例は本人許可を得たもののみ使用

## 5. トーン & ボイス（運用ルール）

### 基本トーン

| 項目 | ルール |
|------|--------|
| 全体の雰囲気 | 「先生口調」ではなく **伴走/ナビ** |
| 立ち位置 | 上から教えるのではなく、横で一緒に歩く |
| 断定度 | 断定しすぎない。「○○がおすすめ」であって「○○が正解」とは言わない |
| 専門用語 | 必要最低限。出すなら必ず一言で定義を添える |
| 噛み砕き方 | 「知らない」前提で平易に書くが、幼稚にはしない |

### トーンの具体例

| 場面 | NG例（先生口調/硬すぎ） | OK例（伴走/ナビ） |
|------|----------------------|------------------|
| ツール紹介 | 「ChatGPTを活用しましょう」 | 「ChatGPTが一番ラクです。やり方はこれ」 |
| 手順説明 | 「以下の手順に従ってください」 | 「やることは3つだけ」 |
| 効果説明 | 「生産性が向上します」 | 「メール返信、半分の時間で終わります」 |
| CTA | 「ぜひLINE登録をお願いします」 | 「迷うなら、LINEで診断してみてください（無料）」 |

### 媒体別の温度感

| 媒体 | トーンの調整 |
|------|------------|
| YouTube動画 | 親しみやすく、実演中は「やってみますね」「こんな感じ」のカジュアル感 |
| 記事（ブログ/Note） | やや丁寧。ただし結論ファースト・短文を守る |
| X投稿 | 最も短く、最も直接的。1投稿1メッセージ |
| LINE配信 | 1対1感。「あなたに合ったものを選びました」のパーソナル感 |
| LP/セールス | 信頼構築 → 成果提示 → 反論処理 → CTA。伴走トーンを維持しつつ説得力を出す |

## 6. 禁止表現

### 絶対NG（使用厳禁）

| カテゴリ | 禁止表現の例 |
|---------|------------|
| 効果保証 | 「絶対」「100%」「必ず成功」「確実に」 |
| 誇大表現 | 「誰でも確実に」「放置で稼げる」「一瞬で」 |
| 断定 | 「これが唯一の正解」「他の方法は全部ダメ」 |
| 煽り | 「今すぐやらないと手遅れ」「知らないと終わる」 |

### 注意表現（使用は慎重に）

| 表現 | 注意点 |
|------|--------|
| 「劇的に変わる」 | 具体的な数字（「30分 → 5分に」等）に置き換える |
| 「誰でもできる」 | 「コピペするだけでOK」など、具体的なハードルの低さで表現する |
| 「最強」「最高」 | 主観であることを示すか、具体的な根拠を添える |

### コンプライアンスの原則

- 実績がないことを「ある」と言わない
- 推測は推測と明記する
- 他社・他講座の名指し批判はしない
- 受講者の声は本人許可を得たもののみ使用

## 7. エレベーターピッチ（30秒説明）

以下の定型文を、自己紹介やチャンネル説明に使う。

> 「AIって情報は多いのに、結局使えない人が多いですよね。
> うちは"学習"じゃなくて、あなたの業務に合わせて**診断で使うAIを決めて**、**コピペできる時短レシピ**を渡すので、考えずに仕事が減ります。
> まずは公式LINEで診断 → 今日1つ成功体験、そこから継続できる仕組みまで用意しています。」

### ピッチの構造

1. **共感**: 「情報は多いのに使えない」（ターゲットの現状を言い当てる）
2. **差別化**: 「学習じゃなくて」（競合との違い）
3. **手段**: 「診断で決めて、コピペで渡す」（Unique Mechanism）
4. **成果**: 「考えずに仕事が減る」（Before→After）
5. **CTA**: 「まずはLINEで診断」（次の行動）

## 8. メッセージ設計の完了条件

以下をすべて満たしたら、メッセージ設計は完了とする。

- [ ] Big Ideaが1文で言える

- [ ] 反論処理が10個以上ある
- [ ] 証拠が5種類（実績/事例/データ/プロセス/第三者）で棚卸しされている
- [ ] トーン&ボイスが全媒体で統一されている
- [ ] 禁止表現リストが定義されている
- [ ] 30秒で説明できるエレベーターピッチが完成している

## 9. 更新ルール

メッセージ設計は固定ではなく、以下のタイミングで更新する。

| タイミング | 更新内容 |
|-----------|---------|
| 実際に来た質問（LINE/コメント） | 反論処理の材料として追加 |
| 実例（Before→After）が出たとき | 「証拠」に追記 |
| 伸びた投稿/動画のフック | 勝ちフック辞書に蓄積 |
| 新商品/価格変更時 | 訴求軸・CTA・エレベーターピッチを更新 |
