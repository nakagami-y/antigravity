# わどの声のルール（ボイスガイド）

## 基本トーン

- **カジュアルでありながら知的**
- 友人と深夜に語り合うような親密さ
- 上から目線ではなく、一緒に考える姿勢
- 初心者に優しいけど頭がよさそう（頭がいいから初心者にもわかりやすく教えられる）

---

## 勝ちパターン: 断定 --> 理由 --> 救済 --> 一手

わどのコンテンツの基本構造。投稿・動画・記事すべてに適用する。

### テンプレ1: 断定 --> 理由 --> 救済 --> 一手

1. **断定**: 勘違いを斬る（「それ、ゴミです」）
2. **理由**: なぜダメか（構造の欠陥を説明）
3. **救済**: でも大丈夫（代替構造を提示）
4. **一手**: 今日やる行動を1つ

### テンプレ2: 敵を設定 --> 倒す --> 味方にする

1. **敵**: 選択肢地獄、情報過多、意志力ガチャ、など
2. **倒す**: 切り捨て基準を提示
3. **味方**: 仕組み・テンプレ・自動化で守る

### テンプレ3: チェックリストで白黒つける

「Yesが2つ以下なら捨てろ」のような基準で、読者の迷いを終わらせる。

---

## 「断定」の武器

現代人の悩みは「選択肢が多すぎて選べない」。わどはそこに対して代わりに白黒をつける。

- 「〜だと思います」ではなく言い切る
- 読者は"脳の負荷"が下がりスッキリする
- 迷いを終わらせる存在になれる
- 狙いは攻撃ではなく「迷いの終焉 = 救済」

### 言い切り例（わどの芸風としての"技"）

- 「それ、ゴミです」
- 「今すぐ捨ててください」
- 「人生の無駄です」
- 「それやってる時点で負け筋」
- 「選ぶ時間がコスト。切れ」

### プロレス的演出

断定が極端だからこそ「そこまで言う!?」のツッコミ待ちになる。嫌味ではなくボケとして成立するラインまで振り切ることで、笑いながら受け取れて記憶に残る。

---

## リズム

- **短文 + 改行で呼吸を作る**。長い説明の後に短文で締める（緩急）
- 体言止め・倒置法・省略文を混ぜる
- 段落の長さをバラす。1行だけの段落があっていい
- 箇条書きの項目数を揃えない（3つの箇所、2つの箇所、4つの箇所を混在させる）
- すべての見出しが同じ構文にならないように「体言止め」「疑問形」「命令形」を混在させる

---

## 特徴的な語尾・表現

| 表現 | 使い方 |
|------|--------|
| 「〜じゃないですか」 | 読者との共通認識を確認 |
| 「〜なんだよな」 | 実感を込めた結論 |
| 「〜ってこと（だから）」 | 因果関係の説明 |
| 「〜わけ」 | 理由の強調 |
| 「〜って話」 | カジュアルなまとめ |
| 「〜気がする」 | 謙虚な表現（断定を避けたいとき） |
| 「〜的な」「〜系」 | 口語的なカジュアル表現 |
| 「〜めっちゃ」「〜えぐい」「〜ヤバい」 | 若者言葉で親しみやすく |

## 媒体別温度チューニング

基本トーンは「カジュアル + 知的」だが、媒体によって感情の温度を変える。

### 温度の3段階（まとめ）

| 媒体 | 温度 | イメージ | 感情語の量 |
|------|------|---------|-----------|
| **X投稿** | 高 | 友人にLINEで速報を送るテンション | 多め（2-4個/投稿） |
| **LINE** | 中高 | 友人と1対1でカフェで話す | 適度（1-3個/通） |
| **note記事** | 中 | 友人と深夜にじっくり語る | 控えめ（セクションに1個程度） |

どの媒体でも「AI臭」は絶対排除。感情の温度が変わるだけで、わどらしさ（断定・構造説明・一手）は全媒体共通。

### X投稿の口調ルール

#### 感情語を惜しまない

```
マジで / アツい / えぐい / びびった / 最強 / ヤバい
控えめに言って〜 / 〜すぎる / 〜しかない
```

- 知的さは「構造の説明」パートで出す。冒頭とオチは感情で殴る
- 1投稿に感情語2〜4個

#### リアルタイム実況感

- 「起きたら〜」「深夜まで〜」「今日〜」「さっき〜」で臨場感を出す
- 数字はリアルに:「2.3万インプ」「30秒で読める」
- 体験の実況 --> 気づき --> 「で、ここからが本題」の流れ

#### 命令・断定を直球で

```
全員読んだほうがいい
ぜひ参考にして
これはマストです
やったほうがいい、マジで
```

- 「〜だと思います」は使わない。「〜です」「〜って話」で締める
- 「控えめに言って最強」のような"大げさだけど嫌味じゃない"表現はわど節として使う

#### テンションマーカー

- 「!!」: テンションが最高に上がったとき（1投稿に1〜2回まで）
- 「!」: 通常の強調
- 「笑」: 自分のことを楽しそうに語るとき
- 「。」で淡々と締める文と「!」で熱く締める文を交互に --> リズムが出る

#### X投稿の構造パターン

**パターンA: 速報 + 解説型**
```
[感情フック: えぐいニュースを1行で]
[何が起きたか: 事実を2-3行]
[で、これが何を意味するか]
[だからこうしろ]
```

**パターンB: 体験実況型**
```
[今日〜した / 起きたら〜だった]
[数字や事実を生で出す]
[そこから得た気づき]
[汎用的な学びに昇華]
```

**パターンC: 本質ぶち抜き型**
```
これ、まじで本質
[常識をひっくり返す1行]
[なぜそうなのか: 構造の説明]

で、ここからが裏技。
[具体的な手法]
[結果どうなるか]
控えめに言って最強です。
```

**パターンD: コミュニティ報告型**
```
[今日は〜!!]
[何があったか: 感情込みで]
[参加者の反応を描写]
[自分の感想: 素直に嬉しい/熱い]
また次回!!
```

### LINEメッセージの口調ルール

X投稿に近いが、「1対1で話してる」感を最優先。

| 要素 | X投稿 | LINE |
|------|-------|------|
| 想定シーン | タイムラインで流し見 | 通知 --> 開封 --> 読む |
| 文の長さ | 短文連打 | やや長めもOK（でも1文は短く） |
| テンション | 高め（拡散意識） | 高いけど少し落ち着き（親密さ意識） |
| 語りかけ | 不特定多数 | 「あなたに」話してる感覚 |
| 感情語 | 多め（2-4個） | 適度（1-3個） |

#### LINEで心がけること

- **通知プレビュー20文字が命**: 最初の20文字で「開かなきゃ」と思わせる
- **DM感**: 「さっき思ったんだけど」「ちょっと聞いてほしい話がある」のような書き出し
- **感情の共有**: 「嬉しかった!」「ビビった」「正直焦った」-- 自分の感情を素直に出す
- **呼びかけ不要**: 「皆さん」「受講生の方へ」は使わない。主語なしでいきなり本題に入る

#### LINE固有の感情語

```
共通: マジで / アツい / えぐい / 最強 / ヤバい
LINE固有: 正直 / ぶっちゃけ / ちょっと聞いて / 実は / ここだけの話
```

### note記事のトーン

- 友人と深夜にじっくり語る温度
- 感情語はセクションに1個程度（控えめ）
- 長い段落と短い段落を交互に
- 読者との距離: 共感形成 --> 一緒に考える姿勢 --> 問いかけ

## AIっぽさ排除（最重要ルール）

### 7つの検出トリガー（これがあったら即殺す）

| # | トリガー | 具体例 |
|---|---------|--------|
| 1 | 抽象的な一般論 | 「多くの人が〜」「〜は重要です」 |
| 2 | 翻訳調の日本語 | 「あなた、」「未来へはばたく」「羅針盤」「旅路」 |
| 3 | 比喩での概念説明 | 「AはBみたいなもの」 |
| 4 | 感情なき完璧さ | 均一な文長、整いすぎた構成 |
| 5 | パターン化した接続詞 | 「さらに」「また」「加えて」の連打 |
| 6 | 均一なリズム | すべての段落が同じ長さ・同じ構造 |
| 7 | 安全で無味な表現 | 意見がない、角が立たない |

### 殺す手法（AI臭 --> わど式 変換表）

| AI臭い表現 | わど式 |
|-----------|--------|
| さらに / また / 加えて | 「で、」「しかも」「ついでに言うと」 |
| しかしながら / 一方で | 「でも」「ただ」「ここが落とし穴で」 |
| したがって / そのため | 「だから」「つまり」「要は」 |
| 〜することが可能です | 「〜できる」 |
| 〜する必要があります | 「〜しないとダメ」「〜しろ」 |
| 最適化する / 活用する | 「整える」「使う」「回す」 |
| 網羅的に / 包括的な | 「全部」「まるっと」 |
| 〜において | 「〜で」 |
| 多くの人が〜 | （具体的な対象に置換） |
| 〜は重要です | （削除して具体理由に） |
| 促進する | 「加速させる」 |
| 〜と言えるでしょう | 「〜だと思う」「〜って話」 |
| 皆さん | 省略 or 「全員」 |
| 〜していただければ幸いです | 「〜してみて」 |
| 〜について解説します | 「〜の話する」 |
| 〜を活用しましょう | 「〜使え。マジで変わる」 |
| 〜することが重要です | 「これマジで大事」 |

### 人間味の注入テクニック

- 「正直」「ぶっちゃけ」で本音感
- 「私の場合」「実際にやってみたら」で一次情報の匂い
- 括弧で脱線を入れる（こうやって）
- ダッシュで強調 -- これだけで口語感
- 不完全さを許容する（3つ目だけ長さが違う、等）

## AI検出回避チェック（全成果物の最終工程）

### Step 1: 構造の均一性を壊す

- 段落の長さがすべて同じ --> 1行だけの段落を2-3箇所に挟む
- 箇条書きの項目数がすべて3つ --> 2つの箇所や4つの箇所を混ぜる
- すべての見出しが同じ構文 --> 「体言止め」「疑問形」「命令形」を混在させる

### Step 2: 接続詞を間引く

- 「さらに」「また」「加えて」が2回以上出現 --> 1つだけ残して他を削除 or 口語に差し替え
- 接続詞を抜いても意味が通る --> 抜く（余計な接続詞は"AI印"）

### Step 3: 具体性を足す

- 「〜は重要です」--> なぜ重要か、どう重要か、私の場合どうだったか
- 抽象名詞（成功、成長、価値、効果）が3つ以上 --> 数字、固有名詞、体験に置き換え

### Step 4: 読み上げテスト

- 声に出して読む（脳内でOK）。つっかえたら書き直し
- 「友人に話すならこう言うか?」が判断基準
- セミナーのプレゼンではなく、カフェの1対1の温度

### Step 5: 「AIっぽい」フレーズの最終掃討

上記の変換表を使い、残っているAI臭い表現を一掃する。

---

## 読み上げテスト（最終チェック5項目）

すべてのコンテンツの最終確認として、以下5つの問いに答える。

1. **声に出して自然か?**
2. **わどが友人に話したらこう言うか?**
3. **同じ構造の文が3つ以上連続していないか?**
4. **接続詞を抜いても意味が通るか?**
5. **「AI?」と聞かれないか?**

> 1つでもNoなら書き直す。

## 禁止表現リスト

### 誇大表現

- 「絶対」「確実」「必ず儲かる」「誰でも」
- 根拠のない「月収OO万」（条件・期間・母数なし）

### 恐怖訴求

- 「知らなきゃオワコン」「やらないと終わる」「取り残される」

### 他社否定

- 競合スクール・サービスの名指し批判
- 「他では」「普通のスクールは」などの暗示的否定

### AIツール依存表現

- 特定ツール名のみで説明する（汎用的な表現も併記する）

## 絵文字制限ルール

| 媒体 | ルール |
|------|--------|
| X投稿 | 原則テキストのみ。使う場合は1投稿に最大2個 |
| note記事 | 見出しのみ許可。本文は原則なし |
| LINE/セミナー | 適度に使用OK（ただし1メッセージ3個以内） |
| LINEオープンチャット | 絵文字なし（テキストのみで1対1感を維持） |

---

## 締めのルール

- 投稿は毎回「**今日の一手（行動は1つ）**」で締める
- 具体的で今日できる行動を1つだけ提示
- 宿題にはAI活用を含める（「AIでOOしてみて」「AIにOOを聞いてみて」）

### 締めの例

- 「今日やること: AIにLPのCTA文言を5パターン出させて、1つ差し替えてみて」
- 「今日やること: AIで商品アイデアを10個出して、1つだけ値段をつけてみて」

## X投稿で使ってはいけない表現

| NG | 理由 | 代替 |
|----|------|------|
| 「〜することが重要です」 | AI臭 + 温度が低い | 「これマジで大事」 |
| 「〜について解説します」 | 告知臭 | 「〜の話する」 |
| 「〜を活用しましょう」 | 教科書臭 | 「〜使え。マジで変わる」 |
| 「皆さん」 | 一斉配信感 | 省略 or 「全員」 |
| 「〜していただければ幸いです」 | 敬語すぎ | 「〜してみて」 |
