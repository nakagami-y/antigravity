## レクチャー 11-1: 30/60/90日ロードマップ（想定時間: 7分）

### 画面構成
- 使用画面: スライド（全編）
- 切替タイミング: なし

### 読み上げ原稿

#### スライド1: セクション11タイトル
最後のセクションです。このセクションでは、作ったテンプレートを「自分の仕事道具」に育てるための90日ロードマップをお伝えします。

「大事なのは作った後、どう続けるか」。このセクションのテーマはそれだけです。

感動的に締めようとはしません。「明日から具体的に何をするか」が決まった状態でこの講座を終えることを目指します。

#### スライド2: 「作って終わりにしない」3ヶ月ロードマップ
テンプレを作って終わりにする人と、テンプレを育てて業務を変える人の差は、最初の3ヶ月の動き方にあります。

大げさな計画は要りません。「1ヶ月目に何をするか」が決まるだけで十分です。

3ヶ月の地図を見ていきましょう。

#### スライド3: 30日目（1ヶ月） — まず1周する
1ヶ月目のゴールは「5つのテンプレを一度使ってみる」ことです。

やることは2つです。

1つ目が「5つの成果物テンプレを実務で1回ずつ使う」こと。週に1つ試せばちょうど1ヶ月です。

2つ目が「使った・使わなかったを記録する」こと。メモ帳でも付箋でも何でもいいです。「議事録は毎週使った。ブログは1回だけ使った」という記録をつけます。

やらなくていいことを言います。テンプレの改善は1ヶ月目にやらなくていいです。新しいスキルを作ることも1ヶ月目には要りません。

「まず動かす」が最初の1ヶ月の全てです。完成度は気にしません。

うまくいかないことも出てくると思います。「SKILL.mdのどこが原因か」と考える習慣だけ作っておいてください。

#### スライド4: 60日目（2ヶ月） — 使ったものを調整する
2ヶ月目のゴールは「よく使うテンプレを1つ選んで自分仕様に育てる」ことです。

やることは3つです。

1つ目が「1ヶ月の記録を見て、一番使ったものを1つ選ぶ」こと。

2つ目が「SKILL.mdのルールや出力形式を微調整する」こと。「もう少しこうしたい」という箇所を1行直すだけでいいです。

3つ目が「CLAUDE.mdに気づいたことを追記する」こと。「この表現は使わない」「文字数はもう少し短くしたい」など、1ヶ月使って気づいたことを足します。

調整の例を出しますね。「SNSカレンダーのフォーマットをCSVからMarkdownに変えた」「レポートの文字数を500字から800字に変えた」「チェックリストに自社特有の確認項目を追加した」といったイメージです。

「1行書き換えるだけ」の調整がほとんどです。完璧を目指すのではなく、少しだけ自分に合わせる感覚で進めます。

#### スライド5: 90日目（3ヶ月） — 自分のスキルを1つ作る
3ヶ月目のゴールは「この講座にないテンプレを1つ追加する」ことです。

自分の業務から「繰り返し作っているもの」を探します。

毎週書いている週次報告。月に1回出す提案書や見積書。繰り返し送るお礼メールやフォローアップメール。定型のアンケート集計やまとめ。

「繰り返しているもの」が次のスキル候補です。

この講座で作った5つ以外の業務を1つ選んで、同じ5Cの流れで作れます。「Contract（型を決める）→Context（前提をCLAUDE.mdに書く）→Chain（工程を分ける）→Check（品質確認）→Carry（コマンド化）」。この5ステップは、どんな成果物にも使えます。

#### スライド6: 90日ロードマップ（まとめ）
3ヶ月を一目で見ると、こうなります。

1ヶ月目は「5つを1回ずつ使う。まず動かす」。

2ヶ月目は「よく使ったものを1つ調整する。SKILL.mdとCLAUDE.mdを微調整」。

3ヶ月目は「自分のスキルを1つ作る。業務の繰り返し作業をテンプレ化」。

3ヶ月後には「テンプレが自分の仕事道具」になっています。大きな変化ではなく、小さな積み重ねです。

#### スライド7: まとめ + 次へ
11-1のまとめです。

- 1ヶ月目はまず動かすことだけ意識します
- 2ヶ月目に1つだけ調整します
- 3ヶ月目に自分のスキルを1つ追加します

次は「自分の仕事に置き換えるワーク」です。ここで一度動画を止めて、実際に書いてもらいます。

### このレクチャーのポイント
- 「3ヶ月で全部変わる」という大げさな約束はしない。「小さな積み重ね」を強調する
- 「1ヶ月目にやらなくていいこと」を言うことで不安を取り除く
- 「繰り返しているもの」を具体例で出して、受講者が自分の業務に当てはめやすくする
- つまずきやすい点: 60日目の「調整」が難しく感じられやすい。「1行書き換えるだけ」という具体例を複数出す

## レクチャー 11-2: 自分の仕事に置き換えるワーク（想定時間: 5分）

### 画面構成
- 使用画面: スライド（全編）
- 切替タイミング: なし

### 読み上げ原稿

#### スライド1: ワーク導入
ここでワークをします。

一度、動画を止めてください。

このまま講座を最後まで見た後、「明日から何をするか」が決まっていないと、何もしない可能性が高いです。今ここで5分使って書いておくだけで、明日実際に動く確率が大きく上がります。

5分で書けます。全部埋まらなくて大丈夫です。

止めましたか？それでは始めます。

#### スライド2: ワークシート
5つの成果物を自分の業務に当てはめます。

次の表を紙かメモ帳に書いてみてください。

```
成果物 | あなたの業務での用途 | 最初に試す日 | 誰に見せる

SNS投稿カレンダー  |  |  |
ブログ下書き       |  |  |
週次・月次レポート  |  |  |
議事録 → タスク化  |  |  |
告知LP            |  |  |
```

「あなたの業務での用途」の列を先に埋めてください。30秒で1つずつ考えます。

「SNS投稿カレンダー」は使わないな、と思ったら空欄で構いません。「議事録 → タスク化」は毎週使えそう、と思ったら「毎週の定例MTG後に使う」と書いてください。

「最初に試す日」には具体的な日付を書きます。「今月中に」ではなく「3月10日」と書きます。

「誰に見せる」には、完成したものを見せる相手を書きます。「上司」でも「チームメンバー」でも「自分だけ」でもいいです。誰かに見せる予定があると、実際に動く確率が上がります。

書けましたか？全部埋まらなくても大丈夫です。1つでも書けていれば十分です。

#### スライド3: 最初の1つを選ぶ
書き終わったら、1つを選んでください。

「明日、これを使ってみる」と言えるもの。今週中に試せる規模のもの。誰かに見せる予定があるものです。

1つに絞ります。2つでも3つでもなく、1つです。

次の文を埋めてください。

「私が最初に試すのは ___（成果物名）___ を使って、___（何を）___ を作ります。試す日は ___ 月 ___ 日です。見せる相手は ___（名前か役職）___ です。」

この1文が書けた人は、この講座を「動いた」に変えられます。書けなかった人も、動画が終わった後でいいので書いてみてください。

#### スライド4: まとめ + 次へ
ワークが終わりました。

書けた方も、まだ書けていない方も大丈夫です。次の最後のレクチャーで講座全体を締めて終わります。

### このレクチャーのポイント
- 「動画を一時停止して書いてください」と明確に指示する
- ワークの後に「書けた人は素晴らしい」と声をかける
- 「全部埋まらなくてもいい」を2回以上言う
- 「1つに絞る」という動作を具体的に促す
- つまずきやすい点: 「誰に見せる」という列が後回しにされやすい。この列があることで他者を巻き込む意識が生まれると補足する
- 撮影時: ワーク中は少し間を置く。受講者が書く時間を意識して、1〜2秒沈黙を作る

## レクチャー 11-3: おわり（想定時間: 3分）

### 画面構成
- 使用画面: スライド（全編）
- 切替タイミング: なし

### 読み上げ原稿

#### スライド1: テンプレ・サンプルの受け取り方
最後に配布資料の案内をします。

この講座で使ったファイルを配布しています。4種類あります。

1つ目は「CLAUDE.mdテンプレート」。コピーしてすぐ使えるベースです。

2つ目は「SKILL.mdサンプル」。5成果物分を揃えています。

3つ目は「チェックリストPDF」。印刷して手元に置けます。

4つ目は「セクション別プロンプト集」。この講座で使った指示文を全部まとめています。

受け取り方はUdemy講座ページの「講座リソース」のタブからダウンロードできます。特にCLAUDE.mdテンプレートは、コピーして自分の名前と職種を書き換えるだけで使えます。今日中にダウンロードしておいてください。

#### スライド2: 次のステップ（優先順位順）
この講座の後にやることを4つ挙げます。優先順位順です。

1番は「配布資料をダウンロードする」。これは今日中にやってください。

2番は「CLAUDE.mdを自分のPCに置く」。今週中を目標にします。

3番は「5つのうち1つを実際の業務で使う」。今週から来週の間です。ワークで書いた日付通りに動いてください。

4番は「使った感想をレビューに書く」。これは任意です。ですが、他の受講者の参考になりますし、書くことで自分の学びが整理されます。

もっと学びたい場合は、Claude Codeの公式ドキュメント（code.claude.com/docs）をお勧めします。日本語版もあり、最新の機能が全部掲載されています。この講座の続編も準備中です。

#### スライド3: 5Cメソッド 最終まとめ
この講座で学んだ5つの型を最後にもう一度確認します。

1つ目のContractは「出力の約束」。ゴールを先に決めることです。何を出したいかを決めてから作業を始めます。

2つ目のContextは「前提固定」。CLAUDE.mdで毎回同じ前提に揃えることです。一度書けば、以降の全作業に反映されます。

3つ目のChainは「役割分担」。分業で精度を上げることです。一気にやらせず、工程を分けて1つずつ進めます。

4つ目のCheckは「品質保証」。最後のOKは人が出すことです。AIの出力をそのまま使わない習慣です。

5つ目のCarryは「再利用」。スキル化して資産にすることです。今日作ったものを次も使える形で残します。

この5つの型は、今後どんな成果物を作るときにも使えます。新しい業務が出てきても、Contractから始めれば道に迷いません。

#### スライド4: 「使うもの」として手元に残す
最後に1つだけ伝えます。

この5Cは「覚えるもの」ではなく「使うもの」です。

困ったときに「今5Cのどこにいるんだっけ」と考える習慣が、この講座の一番の成果です。

配布資料の5C早見表を手元に置いておくと、迷ったときに引き出しやすいです。

#### スライド5: ありがとうございました
道具が揃いました。あとは使うだけです。

この講座で手に入れたのは3つです。

5つの成果物テンプレート。5Cメソッドという判断基準。90日ロードマップという続け方の地図。

完璧を目指さず、まず1つ動かしてみてください。

ご受講ありがとうございました。

### このレクチャーのポイント
- 感動的に締めようとしない。淡々と、しかし温かく話す
- 「ありがとうございました」の前に「まず1つ動かしてみてください」という実用的な一言を必ず入れる
- 5Cの最終まとめは1Cずつ丁寧に読み上げる。流さない
- 「覚えるものではなく使うもの」という言葉を締めの中に入れる
- つまずきやすい点: 最後に余韻を作ろうとして話が長くなりやすい。3分で終わらせることを意識する
- 撮影時: 最後のスライドで名前を言って終わる。「中井でした」など

## レクチャー 11-4: ボーナスレクチャー（想定時間: 3分）

### 画面構成
- 使用画面: スライド（全編）
- 切替タイミング: なし

### 読み上げ原稿

#### スライド1: お疲れさまでした
ここまで完走された皆さん、おめでとうございます。5つの成果物と5つの型、すべて手を動かして体験していただきました。

この講座で学んだ5Cメソッドは、実はもっと応用が利きます。提案書、マニュアル、お礼メール、アンケート集計…。「繰り返す作業」があれば、すべてSkills化できます。

#### スライド2: 次のステップ
最新の活用事例やTips、質問への回答は、わどのLINE公式アカウントで発信しています。

受講者限定の特典として、この講座で使った5Cメソッドの早見表と全プロンプト集をまとめたPDFをご用意しました。LINE公式アカウントに登録いただくと、すぐにダウンロードできます。

続編コースも準備中です。今回の5つの成果物をさらに発展させた「チーム活用編」「自動化パイプライン編」を予定しています。LINE公式アカウントでリリース情報をお届けしますので、ぜひ登録しておいてください。

#### スライド3: 最後のお願い
最後に1つだけお願いがあります。

この講座が役に立ったと感じたら、レビューを書いていただけると嬉しいです。「どの成果物が一番使えた」「こんな場面で役立った」など、あなたの声がこれからの受講者の参考になります。

質問や相談は Q&A セクションからお気軽にどうぞ。

またどこかでお会いしましょう。わどでした。ありがとうございました。

### このレクチャーのポイント
- 過度な売り込みは低評価レビューに直結するため、「自然な次のステップ」として提示
- LINE公式アカウント登録は「受講者限定特典のダウンロード」という明確な理由で誘導
- 続編コースの予告は具体的なタイトルを出して期待感を作る
- レビュー依頼は「どの成果物が使えたか」という具体的な書き方を示唆
- 3分以内厳守。温かく余韻を残すトーンで締める
- Udemy規約: ボーナスレクチャーはコースの最後に1本のみ許可
