## レクチャー 2-1: Claude Codeを入れる（想定時間: 10分）

### 画面構成
- 使用画面: スライド → ターミナル画面 → スライド
- 切替タイミング:
  - 「インストール手順」スライドの後にターミナルへ切替
  - バージョン確認が完了したらスライドへ戻る

### 読み上げ原稿

#### スライド1: レクチャー導入
「AIって難しそう」「プログラミングが必要なんでしょ？」そう思っていませんか？ このセクションが終わる頃には、日本語を入力するだけでファイルが自動で作れるようになります。コードは1行も書きません。

セクション2「準備」に入ります。このセクションでは Claude Code をインストールして、日本語で指示するだけでファイルが作れる状態にします。完成すると、「自分のPCでAIが動いた」という成功体験を持てます。

このレクチャーが終わったときのゴールは3つです。Claude Codeがインストールされている。バージョンを確認できる。アップデートの方法を知っている。この3つです。

#### スライド2: Claude Codeとは？
まずClaude Codeが何かを簡単に説明します。

Claude Codeは、Anthropicが提供するAIコーディングアシスタントです。でも「コーディング」という名前がついていますが、コードを書かなくても使えます。

テキストファイルを作る、内容を整理する、CSVを出す。こういった作業をターミナルから日本語で指示するだけでできます。コードを書く必要はありません。

#### スライド3: インストール手順
では入れていきましょう。

> **[画面切替: ターミナル]**

画面をターミナルに切り替えます。

Macの方はSpotlight検索（Command + Space）で「ターミナル」と入力して開いてください。Windowsの方はスタートメニューから「コマンドプロンプト」または「PowerShell」を開いてください。

では今ここに以下のコマンドを入力します。

```
curl -fsSL https://claude.ai/install.sh | bash
```

curlというのはURLからファイルをダウンロードするコマンドです。今は気にしなくて大丈夫です。「このコマンドでインストールできる」とだけ覚えてください。

Homebrewが入っている方は `brew install --cask claude-code` でも入れられます。Windowsの方は `winget install Anthropic.ClaudeCode` を使います。

コマンドを入力したらEnterキーを押します。

（インストール実行中）

インストールが始まりました。1分から2分かかります。画面にいろいろと文字が出てきますが、最後まで止まらずに動いていれば問題ありません。エラーが出た場合は、後ほど確認します。

期待される出力:
```
Claude Code has been installed successfully!
```

インストールが完了しました。では次にバージョンを確認します。

#### ハンズオン: バージョン確認
次のコマンドを入力します。

```
claude --version
```

期待される出力:
```
claude 1.x.x
```

バージョン番号が表示されれば、インストール成功です。

#### スライド4: アップデートの方法
Claude Codeは定期的に更新されます。curlでインストールした場合は、バックグラウンドで自動的に最新版に更新されます。

手動で更新したい場合は、インストールと同じコマンドを再実行するか、Homebrewの場合は次のコマンドを使います。

```
brew upgrade claude-code
```

「自動でアップデートされるのか？」という質問が来ることがありますが、curlインストールなら自動更新されます。Homebrewの場合は手動でbrew upgradeを実行する必要があります。

> **[画面切替: スライド]**

#### スライド5: まとめ + 次へ
まとめます。

- `curl -fsSL https://claude.ai/install.sh | bash` でインストールします
- `claude --version` でバージョンを確認できます
- curlインストールならバックグラウンドで自動更新されます

次はログインして動くか確認します。

### このレクチャーのポイント
- 「curlって何ですか？」への回答: 「URLからファイルをダウンロードするコマンドです。MacやLinuxには最初から入っています」
- つまずきやすい点: curlコマンドが長いので入力ミスが起きやすい。画面に大きく映して、コピペを推奨する
- 補足: Node.jsは不要。npm方式は公式の推奨ではなくなったため案内しない
- 撮影時: コマンドを入力するときは画面に大きく映す。手元が見えるようにする

  - 認証完了後ターミナルに戻る
  - ファイル作成確認後にスライドへ切替

### 読み上げ原稿

#### スライド1: レクチャー導入
このレクチャーのゴールは3つです。Claudeアカウントでログインできている。簡単なファイルを1つ作れる。「動いた」という成功体験を得る。

特に3つ目が大事です。このレクチャーが終わった後に「自分でもできる」という感覚を持ってもらうことが目的です。

#### ハンズオン: ログイン
では実際にログインしてみましょう。

> **[画面切替: ターミナル]**

ターミナルで次のコマンドを入力します。

```
claude
```

期待される出力:
```
Welcome to Claude Code!
Open the following URL to complete authentication:
https://claude.ai/auth/...
```

ブラウザが自動で開きます。すでにclaude.aiにログインしている場合は、自動で認証が完了します。ログインしていない場合は、メールアドレスとパスワードを入力してください。

（ブラウザでログイン）

ブラウザで「認証が完了しました」と表示されたら、ターミナルに戻ります。

ターミナルに戻ると、Claudeのプロンプトが表示されています。これでログイン完了です。

#### ハンズオン: 動作確認
ではテストしてみます。今ここに次のように日本語で入力します。

```
「hello.txt」というファイルを作って、
中に「動いた！」と書いてください
```

期待される出力:
```
hello.txt を作成しました。
内容: 動いた！
```

（実行完了）

ファイルが作られました。フォルダを確認すると「hello.txt」というファイルができています。

これでClaude Codeが動くことが確認できました。日本語で指示を入れるだけで動きます。これが基本の使い方です。

> **[画面切替: スライド]**

#### スライド1: まとめ + 次へ
まとめます。

- `claude` コマンドで起動、ブラウザでログインします
- 日本語で指示を入れるだけで動きます
- 実はここまでできたら、セクション4のSNSカレンダー作成に先に進んでも大丈夫です

VS CodeとGitは後からでもOKです。早く成果物を作りたい方は、セクション4に進んでから戻ってきても構いません。次はVS Codeを入れて、ファイルを見やすくします。

#### スライド1: レクチャー導入
VS Codeを入れます。VS Codeは「ファイルを見るための道具」です。テキストエディタよりも便利なファイル管理ツールとして使います。

## レクチャー 2-4: Gitを入れる（想定時間: 5分）

### 画面構成
- 使用画面: スライド → ターミナル画面 → スライド
- 切替タイミング:
  - 「最初の3コマンド」スライドの後にターミナルへ切替
  - git diff を見せた後にスライドへ戻る

### 読み上げ原稿

#### スライド1: Gitとは？
Gitを入れます。Gitとは何かを説明します。

ゲームのセーブ機能、イメージできますか？ゲームでは途中でセーブしておくと、ミスしても戻れますよね。Gitはファイルに対して同じことができます。

「元に戻せるセーブポイント」を作る仕組みです。

ファイルを壊してしまっても戻せます。「どこで変わったか」が記録されます。Claude Codeが変更したファイルを後で確認するときにも使います。

Ctrl+Zでは取り消せない大きな変更でも、Gitがあれば戻せます。

#### スライド2: 最初の3コマンド
覚えるコマンドは3つだけです。

> **[画面切替: ターミナル]**

では実際にやってみます。ターミナルで作業フォルダに移動します。

1つ目のコマンドです。

```
git init
```

期待される出力:
```
Initialized empty Git repository in /Users/.../project/.git/
```

「このフォルダをGit管理下に置く」コマンドです。最初の1回だけ実行します。

2つ目のコマンドです。

```
git add .
```

「変更したファイルを次のセーブ対象に入れる」コマンドです。ドット（.）は「全部のファイル」という意味です。

3つ目のコマンドです。

```
git commit -m "最初のコミット"
```

期待される出力:
```
[main (root-commit) abc1234] 最初のコミット
 1 file changed, 1 insertion(+)
 create mode 100644 hello.txt
```

「セーブポイントを作る」コマンドです。`-m` の後ろに書くのは「何を変えたか」のメモです。

#### ハンズオン: git diff で変更確認
次にファイルを少し変更してみます。hello.txtに一行追加します。

```
これはテストです
```

では変更が見えるか確認します。

```
git diff
```

期待される出力:
```
diff --git a/hello.txt b/hello.txt
index ...
+++ b/hello.txt
@@ -1 +1,2 @@
 動いた！
+これはテストです
```

緑色で「+これはテストです」と表示されました。これが追加された行です。赤色で表示されるのは削除された行です。

Claude Codeがファイルを変更したときに、何を変えたかをこのコマンドで確認できます。

> **[画面切替: スライド]**

#### スライド3: まとめ + 次へ
まとめます。

- `git init` でセーブ機能を有効にします（最初の1回だけ）
- `git add .` と `git commit` でセーブポイントを作ります
- `git diff` で変更内容を確認できます

Gitは難しそうなイメージがあるかもしれませんが、この講座ではこの3つだけ使います。エンジニアが使うような高度な使い方は一切出てきません。

次のセクションでは「作り方の土台」として、CLAUDE.mdという設定ファイルを作ります。今回学んだ「3つだけ覚える」の考え方が、そのまま使えます。ちょっと休憩を入れて、次に進みましょう。

### このレクチャーのポイント
- 「ゲームのセーブ機能」という例えを強調する
- `git init` は最初の1回だけ、を繰り返す
- つまずきやすい点: Gitが入っていないMacでは「xcode-select --install」が必要な場合がある。「Gitが入っていない場合はこのコマンドを実行してください」と案内する
- 撮影時: `git diff` の出力は緑・赤の色が見えるようにターミナルを大きく映す
