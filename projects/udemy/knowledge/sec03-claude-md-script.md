## レクチャー 3-1: 前提メモを作る — GEMINI.md（想定時間: 6分）

### 画面構成
- 使用画面: スライド → VS Code画面（またはターミナル） → スライド
- 切替タイミング:
  - 「テンプレート」スライドの後にVS Codeへ切替

### 読み上げ原稿

#### スライド1: GEMINI.mdとは
Google AntiGravityに毎回「30代向けに書いて」「カジュアルなトーンで」と繰り返し伝えていませんか？ 毎回同じ前提を説明し直すのは、かなり非効率です。1つのファイルを置くだけで、それが全部不要になります。

セクション3に入ります。このセクションでは「作り方の土台」を作ります。完成すると、Google AntiGravityが毎回同じルールで動いてくれるようになります。キーワードは「引き継ぎメモ」です。

まずGEMINI.mdを作ります。GEMINI.mdとは何かを説明します。

新しいアルバイトを雇ったとき、最初に引き継ぎメモを渡しますよね。「うちのお店ではこういうルールで動いてほしい」というメモです。

GEMINI.mdはGoogle AntiGravityへの引き継ぎメモです。プロジェクトフォルダに1つ置くだけで、Google AntiGravityが毎回このメモを読んで同じ前提で動いてくれます。

毎回「30代女性向けに書いてください」と入力し直す必要がなくなります。

5つの型の「前提メモ」がこれにあたります。

#### スライド2: GEMINI.mdに書くこと
GEMINI.mdに書くのは3つだけです。これ以上は後から追加できますから、最初は最小限で大丈夫です。

1つ目は「このプロジェクトの目的」です。例えば「30代会社員向けにSNS投稿を月1回作る」のように1文で書きます。

2つ目は「やらないこと」です。例えば「競合サービスとの比較は書かない」「価格情報は含めない」のように書きます。Google AntiGravityが余計なことをしないようにする歯止めです。

3つ目は「よく使う用語」です。例えば「読者＝主にXで情報収集する30代会社員」のように、このプロジェクト固有の定義を書きます。

完璧なGEMINI.mdを作ろうとしないでください。最小限の3つから始めて、後から育てれば大丈夫です。

#### スライド3: GEMINI.mdのテンプレート
テンプレートを見てみましょう。

```markdown
# このプロジェクトについて

## 目的
30代会社員向けにSNS投稿・ブログ記事を月1回作る

## やらないこと
- 競合サービスとの比較は書かない
- 価格情報は含めない

## 用語
- 読者：主にTwitter/Instagramで情報収集する30代会社員
- NG語：「簡単に」「誰でも」「絶対に」
```

このまま真似して作れます。自分のビジネスに合わせて文章を変えてください。

> **[画面切替: VS Code]**

では実際に作ってみます。ここではGoogle AntiGravityを使ってGEMINI.mdを作ります。Google AntiGravityに指示を入れます。

```
GEMINI.mdを作ってください。
以下のセクションを入れてください:
- 目的
- やらないこと
- 用語

内容はSNS投稿カレンダー作成プロジェクト向けで書いてください。
```

期待される出力:
```
GEMINI.md を作成しました。

# このプロジェクトについて

SNSで定期的に投稿するためのカレンダーを毎月作成する

## やらないこと

- 根拠のない数字・統計は含めない

## 用語
- 投稿者：このアカウントの運営担当者
- NG語：「絶対に」「必ず」「誰でも」
```

作られました。内容を確認して、自分のプロジェクトに合わせて修正します。VS Codeのファイル一覧にGEMINI.mdが追加されていますので、クリックして内容を確認・編集できます。

> **[画面切替: スライド]**

#### スライド4: まとめ + 次へ
まとめます。

- GEMINI.mdはプロジェクトフォルダに1つ置くだけで自動で読み込まれます
- 目的・やらないこと・用語の3つから始めます
- 後から追記すれば育っていきます

GEMINI.mdは作って終わりではなく、使いながら育てるものです。最初は3行でも大丈夫です。次は「出力の形」を決めます。

### このレクチャーのポイント
- 「新しいアルバイトへの引き継ぎメモ」という例えを使う
- 「完璧なものを作ろうとしないで」を強調する
- つまずきやすい点: GEMINI.mdのファイル名は大文字・小文字が決まっている（GEMINI.md）。小文字のclaude.mdでは読み込まれないことを注意喚起する

## レクチャー 3-2: 出力の形を決める — チェックしやすいテンプレ（想定時間: 5分）

### 画面構成
- 使用画面: スライド → VS Code画面 → スライド
- 切替タイミング:
  - 「テンプレート」スライドの後にVS Codeへ切替
  - GEMINI.mdへの追記を実演後にスライドへ戻る

### 読み上げ原稿

#### スライド1: このレクチャーのゴール
このレクチャーのゴールは3つです。出力フォーマットを事前に決められる。チェックする項目を先に整理できる。「合格・不合格」の基準が明確になる。

5つの型でいうと「出力の約束」を実践するレクチャーです。

#### スライド2: 出力フォーマットとは
出力フォーマットとは「こういう形で出してほしい」という指定のことです。

先に決めるものは4つです。

まずファイル形式。CSVか、Markdownか、テキストかを決めます。

次に構成・項目。どんな列やセクションが必要かを決めます。

次に文字数。本文は何文字以内か、タイトルは何文字かを決めます。

最後にトーン。丁寧語なのかカジュアルなのかを決めます。

これを先に決めると、修正が減ります。「なんか違う」が減ります。

#### スライド3: チェック項目を先に決める
次に「チェック項目」を先に決めます。

後からチェック項目を考えると、見落としが出やすいです。作業前に「何を確認するか」を決めておくのがコツです。

SNSカレンダーのチェック項目の例です。

- 同じ日に投稿が重複していないか
- NG表現が入っていないか（「絶対」「誰でも」）
- 文字数が制限内か（Xは140字、Instagramは2200字）
- 曜日がズレていないか

これをGEMINI.mdに書いておくと、毎回有効になります。

#### スライド4: テンプレートをGEMINI.mdに追記
先ほど作ったGEMINI.mdに追記します。

> **[画面切替: VS Code]**

GEMINI.mdを開いて、末尾に以下を追記します。

```markdown
## 出力のルール

### SNSカレンダー
- 形式：CSV（投稿日,曜日,媒体,本文,ハッシュタグ,CTA）
- 1投稿あたり：本文140字以内（X）、2200字以内（Instagram）
- ハッシュタグ：3〜5個

### チェック項目
- [ ] 同じ日に重複投稿がない
- [ ] NG語が含まれていない（「絶対に」「誰でも」「簡単に」）
- [ ] 文字数が制限内である
- [ ] 曜日と投稿日が一致している
```

このように書いておくと、Google AntiGravityが自動でこの形に合わせて出力してくれるようになります。

ファイルを保存します。これで出力ルールの設定が完了しました。

> **[画面切替: スライド]**

#### スライド5: まとめ + 次へ
まとめます。

- 出力形式・文字数・トーンを事前に決めます
- チェック項目は作業前に作ります
- GEMINI.mdに書いておくと毎回有効になります

これで土台が完成しました。GEMINI.mdに目的・やらないこと・用語・出力ルール・チェック項目が入りました。次のセクションからいよいよ成果物を作っていきます。

### このレクチャーのポイント
- 「料理のレシピを先に見てから買い物に行く」という例えが使える
- GEMINI.mdは「作ったら終わり」ではなく「使いながら育てる」を繰り返して伝える
- つまずきやすい点: GEMINI.mdの構造（マークダウンの見出し階層）を間違えると、Google AntiGravityが解釈できない場合がある。「## で始まる見出しを使う」を徹底する
- 撮影時: 追記した後にファイルを保存（Command + S / Ctrl + S）する操作を必ず見せる
