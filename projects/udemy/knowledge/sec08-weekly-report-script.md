#### スライド4: 入力CSVの3つのルール
整えるべきポイントは3つです。

1つ目は列名のルールです。日本語より英語にすること（date、sales、categoryなど）、スペースや全角文字は使わないことが基本です。

2つ目は数値のルールです。カンマ区切りを外すこと（「10,000」→「10000」）、単位は列名に含めること（sales_yen、count_timesなど）です。

3つ目は日付のルールです。YYYY-MM-DD形式に統一します（2026-03-01のように）。

この3つを守るだけで、Google AntiGravityの集計精度が大幅に上がります。

#### スライド5: 修正前 vs 修正後（対比）
実際によくある「ダメなCSV」と「良いCSV」を比べてみます。

修正前はこうなっています。
```
日付,売上,カテゴリ
2026/3/1,¥10,000,A
2026-03-02,8000円,B
```

日付の形式がバラバラ（スラッシュとハイフンが混在）、金額に「¥」や「円」が入っている、列名が日本語です。

修正後はこうなります。
```
date,sales_yen,category
2026-03-01,10000,A
2026-03-02,8000,B
```

列名が英語になり、日付が統一され、数字が素の数値になりました。この修正だけで集計の精度が全然変わります。

#### スライド3: レポートの基本構成（5要素）
レポートは5つの要素で構成します。

2番目が結論です。「一言で言うと何が起きたか」を先に書きます。「先週比で売上が12%増加」のように具体的に書きます。

## レクチャー 8-3: 数字をチェック（想定時間: 12分）

#### スライド1: 8-3レクチャータイトル
このレクチャーがセクション8の核心です。Check（品質保証）を深掘りします。

数字のチェックは、病院の検査と同じです。健康診断で血液検査をしますよね。数字がおかしければ再検査する。「たぶん大丈夫」では済まさない。AIが出したレポートの数字も同じです。合計がズレていないか、前週比が飛びすぎていないか。数字だけは必ず人間が確認します。

レポートで起きやすい数字のミスは4つのパターンがあります。合計ミス、前週差のズレ、外れ値の見落とし、テーブル間の整合性ミス。この4つをGoogle AntiGravityと一緒に確認する方法を学びます。

#### スライド2: このレクチャーで達成すること
1つ目は4つの数字チェックパターンを理解すること。2つ目は各チェックの依頼方法を覚えること。3つ目は結果の見方と対処法を身につけることです。

#### スライド3: 4つのチェックパターン概要（表）
4つのチェックパターンを整理します。

1番目が合計チェック。内訳の和が総計と一致するかを確認します。優先度は高です。

2番目が前週差チェック。前週比の変動幅が妥当かを確認します。優先度は高です。

3番目が外れ値チェック。桁違いや異常な数値がないかを確認します。優先度は中です。

4番目が整合性チェック。複数テーブル間で同じ指標が矛盾していないかを確認します。優先度は中です。

4つのうち「合計」と「前週差」だけでも必ずやってください。この2つをやるだけで大きなミスはほとんど防げます。
