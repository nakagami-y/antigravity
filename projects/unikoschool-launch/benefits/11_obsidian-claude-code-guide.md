## なぜAI秘書に「記憶」が必要なのか

ChatGPTやClaude（Web版）の弱点は「記憶がリセットされること」です。

- 昨日伝えたルールを今日は忘れている
- 先週作った資料の内容を覚えていない
- 毎回「うちの会社は○○で〜」と説明し直す

Claude Code Desktop + Obsidianを組み合わせると、この問題が解決します。

**AI秘書が過去のやり取りを「覚えている」状態が作れます。**

---

## Obsidianとは

Obsidianは無料のメモアプリです。特徴は:

- Markdownファイルをローカルに保存（クラウド不要）
- ファイル同士をリンクでつなげられる（双方向リンク）
- フォルダ構成が自由（自分のルールで整理できる）
- プラグインで機能を拡張できる

**Claude Code Desktopとの相性が抜群な理由:**

Claude Codeはローカルのファイルを直接読み書きできます。Obsidianもローカルにファイルを保存します。つまり、ObsidianのメモをClaude Codeが直接参照・更新できるのです。

## 15分で完了するセットアップ

### Step 1: Obsidianをインストール（3分）

1. https://obsidian.md からダウンロード
2. インストールして起動
3. 「新しいVaultを作成」を選択
4. Vault名を「ai-workspace」、保存先をデスクトップに設定

### Step 2: フォルダ構成を作る（5分）

Claude Code Desktopを開いて、以下を伝えます。

```
デスクトップの ai-workspace フォルダに、以下の構成でフォルダを作ってください。

ai-workspace/
├── CLAUDE.md          ← AI秘書の前提書
├── daily/              ← 日次メモ
│   └── YYYY-MM-DD.md
├── projects/           ← プロジェクト別の資料
├── knowledge/          ← 学んだことの記録
├── outputs/            ← AI秘書の出力物
│   ├── reports/
│   ├── drafts/
│   └── research/
└── references/         ← 参考資料
```

### Step 3: CLAUDE.mdを配置する（5分）

Day 2で作成したCLAUDE.mdを `ai-workspace/CLAUDE.md` に保存します。

```
以下の内容でCLAUDE.mdを作成してください。
[Day 2で作ったCLAUDE.mdの内容を貼り付け]
```

### Step 4: 動作確認（2分）

Claude Code Desktopで以下を実行します。

```
ai-workspace フォルダの構成を確認して、CLAUDE.mdの内容を読み上げてください。
```

正しく表示されれば完了です。

## AI秘書に「記憶」を持たせる3つの方法

### 方法1: CLAUDE.md（長期記憶）

CLAUDE.mdはAI秘書が「毎回自動で読む」ファイルです。

**ここに書くもの:**
- 自分の職業、役職
- 文体のルール
- よく使う用語
- やってほしくないこと

**書かなくていいもの:**
- 今日だけ使う情報（それはdaily/に書く）
- 一時的なプロジェクト情報（それはprojects/に書く）

### 方法2: knowledge/フォルダ（知識の蓄積）

仕事で学んだこと、調べたことをここに保存します。AI秘書に「knowledge/フォルダを参照して」と伝えると、過去の知識をもとに回答してくれます。

```
「先月調べた競合分析の結果をもとに、今月の戦略を提案して」
→ AI秘書が knowledge/competitive-analysis.md を読んで提案
```

### 方法3: daily/フォルダ（日次の文脈）

毎日のメモ、タスク、議事録をここに保存します。

```
「今日のMTGの議事録を daily/2026-03-20.md に保存して」
「昨日の議事録の続きで、決定事項を整理して」
→ AI秘書が昨日のファイルを読んで、文脈を引き継いで回答
```

## 実例: 議事録 → ナレッジ自動蓄積

### 毎日のフロー

```
1. MTGの内容をClaude Codeに伝える
   「今日の会議の内容を整理して」

2. AI秘書が議事録を生成 → daily/2026-03-20.md に保存

3. 議事録から「今後も使える知識」を抽出
   「この議事録から、ナレッジとして保存すべき情報を抽出して」

4. AI秘書がknowledge/に自動保存
   → knowledge/顧客A-要望一覧.md
   → knowledge/競合B-新サービス情報.md
```

### 1ヶ月後の効果

```
「顧客Aの過去の要望をまとめて」
→ AI秘書が knowledge/顧客A-要望一覧.md を参照
→ 1ヶ月分の要望が時系列で整理された一覧が出る

「競合Bの最近の動きを教えて」
→ AI秘書が knowledge/競合B-*.md を参照
→ 過去1ヶ月の情報が集約されて表示される
```

手動でファイルを検索する必要がなく、AI秘書が「覚えている」状態で回答してくれます。

## Obsidian活用のコツ

### 1. 双方向リンクを活用する

Obsidianでは `[[ファイル名]]` と書くと、ファイル同士をリンクできます。

```markdown
# 2026-03-20 議事録

## 参加者
- 田中さん（[[顧客A]]担当）

## 議題
- [[プロジェクトX]]の進捗確認
- [[競合B]]の新サービスへの対応

## 決定事項
- [[プロジェクトX]]の納期を3/31に延長
```

Claude Codeに「プロジェクトXに関連する全ファイルを探して」と伝えると、リンクされたファイルを辿って関連情報を集約してくれます。

### 2. テンプレートを用意する

日次メモのテンプレートを作っておくと、毎日の記録が楽になります。

```
「daily/のテンプレートを作ってください。
 セクション: タスク、MTGメモ、学び、明日の予定」
```

### 3. 定期的にknowledge/を整理する

月に1回、Claude Codeに「knowledge/フォルダを整理して」と伝えます。

```
「knowledge/フォルダの内容を確認して、
 重複している情報をまとめ、
 古くなった情報に[要更新]タグをつけてください」
```

## トラブルシューティング

### 「ファイルが見つからない」と言われる

**原因**: Claude Code Desktopが別のフォルダを見ている
**対処**: 「ai-workspaceフォルダを作業ディレクトリにしてください」と伝える

### 「CLAUDE.mdが反映されない」

**原因**: ファイルの保存場所が間違っている
**対処**: CLAUDE.mdがフォルダの直下（ルート）に置かれているか確認

### 「Obsidianで表示が崩れる」

**原因**: Markdownの書式の問題
**対処**: Claude Codeに「Obsidian互換のMarkdownで書いてください」と伝える

## 次のレベル: 自動化との組み合わせ

Obsidianの記憶とcron自動化を組み合わせると、さらに強力になります。

- 毎朝9時: daily/にタスクテンプレートを自動生成
- 毎日22時: 今日のdaily/を読んで翌日の準備を自動作成
- 毎週月曜: weekly/に1週間の振り返りを自動生成
- 毎月1日: knowledge/の整理提案を自動生成

ユニコスクール本講座では、こうした「記憶 x 自動化」の仕組みを体系的に構築していきます。
