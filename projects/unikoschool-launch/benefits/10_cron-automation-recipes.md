# cron自動化テンプレート集 — 非エンジニア向け3レシピ

> 対象: 講座購入特典（CL期間中購入者限定）
> 配布: 購入直後にLINE配信

---

## cronとは（1分で理解）

cronは「決まった時間に決まった処理を自動実行する仕組み」です。

身近な例でいうと:
- 目覚まし時計 = 毎朝7時にアラームを鳴らす
- 洗濯機の予約運転 = 朝6時に洗濯を開始する

これと同じことを、パソコン上の作業で実現します。

**cronでできること:**
- 毎朝9時にタスク一覧をSlackに自動送信
- 毎週月曜に週次レポートを自動生成
- 毎日22時にSNS投稿の下書きを自動ストック

**実際の効果:**
- 手動40時間/月 → 自動15分/月の事例あり
- 「やり忘れ」がゼロになる
- 起きたらもう仕事が終わっている

## 事前準備

### Claude Code Desktopに頼むだけ

非エンジニアの方に朗報です。cronの設定は自分でやる必要がありません。

Claude Code Desktopに「この作業を毎日9時に自動実行して」と伝えるだけでOKです。以下のレシピは、そのまま伝える台本として使えます。

## レシピ1: 毎朝9時にタスクをSlackに送信

### やりたいこと

毎朝9時に、今日やるべきタスクを自動でSlackに送信する。朝出勤してSlackを開いたら、もうタスク一覧が届いている状態にする。

### Claude Code Desktopへの指示

```
以下の自動化を設定してください。

やること: 毎朝9時に、以下の処理を自動実行
1. タスクリストファイル（tasks.md）を読み込む
2. 期日が今日のタスクを抽出
3. 優先度順に並べ替え
4. 以下のフォーマットでSlackの #daily チャンネルに送信

フォーマット:
---
おはようございます。本日のタスクです。

【高優先度】
- タスク名（期限: HH:MM）

【通常】
- タスク名

残タスク数: X件
---
```

### 出力イメージ

毎朝Slackに以下が届きます:

```
おはようございます。本日のタスクです。

【高優先度】
- A社提案書の最終チェック（期限: 12:00）
- 月次レポートの提出（期限: 17:00）

【通常】
- チームMTGのアジェンダ作成
- 経費精算の申請

残タスク数: 4件
```

## レシピ2: 毎週月曜に週次レポートを自動生成

### やりたいこと

毎週月曜の朝8時に、先週の活動をまとめた週次レポートを自動生成する。上司への報告が1分で終わるようになる。

### Claude Code Desktopへの指示

```
以下の自動化を設定してください。

やること: 毎週月曜日の朝8時に、以下の処理を自動実行
1. 先週月曜〜金曜の作業ログファイルを読み込む
2. 以下のフォーマットで週次レポートを生成
3. reports/ フォルダに保存（ファイル名: weekly-report-YYYY-MM-DD.md）

フォーマット:
---
# 週次レポート（MM/DD〜MM/DD）

## 今週の成果
- [完了したタスクを箇条書き]

## 数値実績
- 商談件数: X件
- 提案書作成: X件
- 成約: X件

## 来週の予定
- [来週のタスクを箇条書き]

## 課題・相談事項
- [あれば記載]
---

作業ログのフォルダ: daily-logs/
```

### 出力イメージ

```
# 週次レポート（3/10〜3/14）

## 今週の成果
- A社向け提案書の完成・提出
- B社との初回商談（好感触）
- 月次レポートの提出

## 数値実績
- 商談件数: 3件
- 提案書作成: 2件
- 成約: 0件

## 来週の予定
- A社フォローアップ面談
- C社への初回アプローチ
- 四半期プレゼン資料の作成開始

## 課題・相談事項
- B社の予算感が不明。ヒアリング方法を相談したい
```

## レシピ3: 毎日22時にSNS投稿下書きを自動ストック

### やりたいこと

毎日22時に、翌日のSNS投稿の下書きを自動生成する。朝起きたら下書きが用意されている状態にする。

### Claude Code Desktopへの指示

```
以下の自動化を設定してください。

やること: 毎日22時に、以下の処理を自動実行
1. 投稿カレンダー（sns-calendar.csv）から明日の予定テーマを取得
2. テーマに基づいて投稿文を生成
3. drafts/ フォルダに保存（ファイル名: draft-YYYY-MM-DD.md）

投稿のルール:
- X向け: 140字以内
- ハッシュタグ: 2-3個
- CTA: 1つ（リンクまたはアクション）
- トーン: [あなたのブランドボイスを記載]

投稿カレンダーの列: 日付, テーマ, ターゲット, CTA

生成パターン:
1案: そのまま使える完成版
2案: 違う切り口のバリエーション

```

## 自動化の始め方（3ステップ）

### Step 1: 自動化したい作業を1つ選ぶ

上の3レシピから1つ選ぶか、自分の仕事で「毎回同じことをやっている作業」を1つ見つけてください。

### Step 2: Claude Code Desktopに伝える

レシピの「Claude Code Desktopへの指示」をそのままコピーして、自分の情報に書き換えて伝えます。

```
「以下の自動化を設定してください」
```

これだけでOK。Claude Codeが:
- 必要なスクリプトを作成
- cron設定を提案
- テスト実行して動作確認

まで全部やってくれます。

### Step 3: 動作確認して微調整

「テスト実行してください」と伝えれば、本番の時間を待たずにすぐ試せます。出力を見て「ここを変えて」と伝えるだけで調整完了。

## よくある質問

**Q: Macじゃないと使えない？**
A: cron自体はMac/Linuxの機能です。Windowsの場合は「タスクスケジューラ」という同等の機能があり、Claude Codeが自動で設定してくれます。

**Q: パソコンの電源を切ったら動かない？**
A: その通りです。cronはパソコンが起動中の間だけ動きます。常時実行したい場合はクラウドサービス（GitHub Actions等）を使う方法があります。これもClaude Codeに「クラウドで実行できるようにして」と伝えるだけです。

**Q: セキュリティは大丈夫？**
A: 自動化の処理は全てあなたのパソコン上で実行されます。外部に情報が送信されるのは、Slack送信やメール送信など「あなたが設定した送信先」のみです。

**Q: 止めたくなったら？**
A: Claude Codeに「この自動化を停止してください」と伝えるだけです。いつでも再開できます。
