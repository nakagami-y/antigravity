## Vibe Codingとは何か

2025年2月、元Tesla AIディレクターのAndrej Karpathyが「Vibe Coding」という言葉を生み出しました。

**定義: コードを書かずに、AIとの会話だけでソフトウェアを作ること。**

従来のプログラミングとの違いはシンプルです。

- **従来**: プログラミング言語を学ぶ → コードを書く → エラーを直す → 完成
- **Vibe Coding**: 「こんなの作って」と日本語で伝える → AIが作る → 「ここ直して」と伝える → 完成

プログラミング知識ゼロでも、会話ができればアプリが作れる。これが2026年最大のトレンドです。

---

## なぜ今、Vibe Codingなのか

1. **Claude Code Desktopの登場**: ファイル操作まで含めた開発をAIが代行できるようになった
2. **精度の飛躍的向上**: 2024年は「おもちゃレベル」→ 2026年は「実用レベル」に
3. **非エンジニアの参入障壁がゼロに**: コードの読み書き不要。日本語の指示だけ

## Claude Code DesktopでVibe Coding: 3ステップ

### Step 1: 作りたいものを言葉にする

Claude Code Desktopを開いて、作りたいものを日本語で伝えます。

```
「毎月の家計を管理できるシンプルなWebアプリを作ってください。
 - 収入と支出を入力できる
 - カテゴリ別に集計される
 - グラフで可視化できる」
```

ポイント: 完璧な仕様書は不要。「こんな感じ」でOK。

### Step 2: 動かしてみて、修正を伝える

AIが作ったものを実際に動かしてみます。「ここが違う」「ここを変えたい」を伝えるだけ。

```
「グラフの色を青系に統一してください」
「カテゴリに"交際費"を追加してください」
「先月との比較ができるようにしてください」
```

### Step 3: 完成したら保存する

満足いくものができたら、そのまま使い始めるだけ。

```
「このアプリをデスクトップからすぐ開けるようにしてください」
```

## 実践例1: 家計簿アプリを会話だけで作る

### 最初の一言

```
家計簿アプリを作ってください。以下の機能が欲しいです。

- 日付、金額、カテゴリ、メモを入力できるフォーム
- カテゴリ: 食費、住居、交通、日用品、交際費、その他
- 月別の支出合計をグラフで表示
- CSVでデータをエクスポートできる
- デザインはシンプルで見やすく
```

### AIとの会話（修正のやり取り）

**あなた**: 「入力フォームが小さすぎます。もう少し大きくしてください」
**AI**: （フォームサイズを調整）

**あなた**: 「カテゴリ別の円グラフも欲しいです」
**AI**: （円グラフを追加）

**あなた**: 「先月のデータと比較できるようにしてください」
**AI**: （比較機能を追加）

実際の所要時間: 約15-20分

## 実践例2: 自分専用タスク管理ツール

### 最初の一言

```
シンプルなタスク管理ツールを作ってください。

- タスクの追加・完了・削除ができる
- 優先度（高/中/低）をつけられる
- 期日を設定できる
- 期日が近いタスクを上に表示
- 完了したタスクは下に移動
- ローカルに保存されるようにする
```

### 会話で機能を追加していく

```
「タスクにタグをつけられるようにしてください（仕事、プライベート、勉強）」
「今日のタスクだけフィルタリングする機能がほしいです」
「完了したタスクの数を週間グラフで見たいです」
```

## 実践例3: 営業日報の自動整形ツール

### 最初の一言

```
営業日報を自動整形するツールを作ってください。

入力: テキストで自由に書いた日報（箇条書きでもOK）
出力: 以下のフォーマットに自動整形

## 本日の活動
- 訪問先と商談内容

## 成果
- 受注金額、アポ件数

## 明日の予定
- 訪問予定

## 所感
- 気づき、改善点
```

これだけで、毎日の日報作成が5分で終わるようになります。

## よくある失敗と対処法

### 失敗1: 「すごいアプリを作って」と漠然と頼む

**問題**: 指示が曖昧すぎて、AIが何を作っていいかわからない

**対処法**: 具体的な機能を3-5個箇条書きにする
```
NG: 「すごい家計簿を作って」
OK: 「月別の支出をグラフで見られる家計簿。カテゴリは食費、住居、交通の3つ」
```

### 失敗2: 最初から完璧を求める

**問題**: 長い仕様書を一度に渡すと、AIが混乱する

**対処法**: まず最小限で作り、会話で少しずつ足していく
```
1回目: 基本機能だけ作る
2回目: デザインを調整
3回目: 追加機能を足す
```

### 失敗3: エラーが出てパニックになる

**問題**: エラーメッセージを見て「壊れた」と思ってしまう

**対処法**: エラーメッセージをそのままAIに伝える
```
「このエラーが出ました。直してください」
（エラーメッセージをコピー&ペースト）
```
AIがエラーの原因を特定して直してくれます。

### 失敗4: 修正のたびにゼロから作り直す

**問題**: 「やっぱり違う」と言うたびに全部作り直している

**対処法**: 「ここだけ変えて」と修正箇所を絞る
```
NG: 「全体的にやり直して」
OK: 「グラフの色だけ青に変えて、他はそのままで」
```

## Vibe Codingの3つのコツ

### 1. 「小さく始めて、育てる」

最初から100点を目指さない。60点のものを作って、会話で80点、90点と育てていく。

### 2. 「見せて、直す」

文章で説明するより、「今の画面」を見せてから修正を伝える方が速い。
Claude Code Desktopならスクリーンショットも理解してくれます。

### 3. 「保存して、再利用する」

うまくいった作り方（プロンプト）は保存しておく。次に似たものを作るときに再利用できます。これが5Cメソッドの「Carry」です。
